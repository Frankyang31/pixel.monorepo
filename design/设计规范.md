# AI 内容生成平台 — 设计规范 v1.0

> 整理于 2026-03-24，作为全站所有页面的设计基准。
> 参考 Leonardo.ai 视觉精髓，差异化演绎，打造独特品牌调性。

---

## 设计定位

**关键词：** 创意 · 专业 · 沉浸 · 克制

不做冷硬的企业 SaaS，也不做廉价的消费级工具。我们的目标用户是创作者——他们需要一个让作品说话、界面退到背景的工具。视觉上要有「深夜工作室」的沉浸感，同时让 AI 生成的内容成为绝对主角。

---

## 一、双主题策略

| 主题 | 适用场景 | 作用 |
|------|----------|------|
| **Light Theme（浅色主题）** | 首页、工具详情页、博客、定价页、活动页等公域营销页 | 建立信任感，便于 SEO 内容阅读，降低转化摩擦 |
| **Dark Theme（深色主题）** | 工作台、作品广场、我的作品集、个人中心等产品内页 | 降低视觉疲劳，突出 AI 生成内容，营造沉浸创作氛围 |

> 认证页（登录/注册）采用深色主题，强化"进入产品"的仪式感。

## 二、色彩系统

### 2.1 核心色板

#### 主色 — 紫蓝渐变（Brand Gradient）

```
--color-brand-start:   #7B5EF0    /* 偏暖的紫色 */
--color-brand-end:     #4F8EF7    /* 明亮的蓝色 */
--color-brand-mid:     #6370F3    /* 渐变中间值，可单独使用 */
```

渐变方向默认：135°（左下 → 右上），即 `linear-gradient(135deg, #7B5EF0, #4F8EF7)`

使用场景：主 CTA 按钮、高亮标签、Progress 进度条、Logo 点缀色

#### 辅助强调色

```
--color-accent-purple:  #9B87F5   /* 浅紫，用于 hover 状态、标签 */
--color-accent-blue:    #60A5FA   /* 天蓝，用于链接、图标高亮 */
--color-accent-teal:    #2DD4BF   /* 青绿，用于成功状态、积分增加 */
--color-accent-amber:   #FBBF24   /* 琥珀黄，用于警告、积分不足提示 */
--color-accent-red:     #F87171   /* 柔红，用于错误状态 */
```

## 三、字体系统

### 3.1 字体族

```css
/* 主字体（正文、UI 标签、按钮） */
--font-sans: 'Inter', 'PingFang SC', 'Noto Sans SC', system-ui, sans-serif;

/* 展示字体（大标题、Hero 区域） */
--font-display: 'Inter', 'PingFang SC', sans-serif;
/* 注：Inter 在大字号下的字重表现已足够优秀，无需单独引入展示字体 */

/* 代码字体（Prompt 输入区、参数展示） */
--font-mono: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', monospace;
```

**中文字体降级策略：** `PingFang SC`（macOS/iOS）→ `Noto Sans SC`（Android/Windows）→ `Microsoft YaHei`（Windows 兜底）

## 五、圆角系统

```css
--radius-none:  0px
--radius-sm:    4px    /* 小标签、徽章 */
--radius-md:    8px    /* 按钮、输入框 */
--radius-lg:    12px   /* 卡片、面板 */
--radius-xl:    16px   /* 大卡片、模态框 */
--radius-2xl:   24px   /* 特殊展示卡片 */
--radius-full:  9999px /* 圆形按钮、头像、Pill 形状标签 */
```

**使用原则：** 组件越大，圆角越大。保持层级一致性——卡片内的按钮圆角应小于或等于卡片本身的圆角。

## 七、动效规范

### 7.1 过渡时长

```css
--duration-instant: 0ms      /* 无动画（can-hover 禁用时） */
--duration-fast:    100ms    /* 微交互：checkbox、toggle */
--duration-normal:  200ms    /* 常规交互：hover、focus */
--duration-slow:    300ms    /* 面板展开、模态框出现 */
--duration-slower:  500ms    /* 页面过渡 */
```

### 7.2 缓动函数

```css
--ease-default:  cubic-bezier(0.4, 0, 0.2, 1)   /* Material Design 标准，大多数场景 */
--ease-in:       cubic-bezier(0.4, 0, 1, 1)      /* 退出动画 */
--ease-out:      cubic-bezier(0, 0, 0.2, 1)      /* 进入动画 */
--ease-spring:   cubic-bezier(0.34, 1.56, 0.64, 1) /* 弹性效果，用于成功反馈 */
```

### 7.3 动效原则

- **功能性优先**：动效服务于信息传递，不为动而动
- **减速感**：进入用 `ease-out`，出场用 `ease-in`，产生真实物理感
- **尊重用户偏好**：检测 `prefers-reduced-motion`，关闭所有非必要动画

## 八、布局系统

### 8.1 栅格

- **桌面端**：12 列栅格，最大宽度 `1280px`，列间距 `24px`，左右边距 `40px`
- **平板端（768–1024px）**：8 列栅格，列间距 `20px`，左右边距 `24px`
- **移动端（< 768px）**：4 列栅格，列间距 `16px`，左右边距 `16px`

### 8.2 工作台布局

```
┌──────────────────────────────────────────────────┐
│  Nav（64px 固定顶部）                              │
├──────────────┬───────────────────────────────────┤
│              │                                   │
│  左侧面板     │  右侧结果区                        │
│  (320–360px) │  (flex-1)                         │
│              │                                   │
│  · 模式切换  │  · 生成中：任务占位卡片             │
│  · Prompt 区 │  · 完成：图片/视频网格展示           │
│  · 参数配置  │  · 支持多任务并行展示               │
│  · 生成按钮  │                                   │
│              │                                   │
└──────────────┴───────────────────────────────────┘
```

- 左侧面板：固定宽度，不随内容滚动（`position: sticky`）
- 右侧结果区：独立滚动，支持无限加载历史
- 移动端：折叠为上下布局，参数区默认收起

## 十、品牌差异化原则（对比 Leonardo.ai）

| 维度 | Leonardo.ai | 我们的方向 |
|------|-------------|------------|
| 主色 | 蓝色（冷） | 紫蓝渐变（带暖意） |
| 背景（深色） | 纯深灰 | 极深紫黑（带微紫调） |
| 背景（浅色） | 纯白 | 极浅暖灰（#F5F4F8） |
| 字体 | 系统字体 | Inter（更统一的跨平台效果） |
| 视觉情绪 | 专业工具感 | 创意工作室感 |
| 图形语言 | 写实截图为主 | 截图 + 抽象光效粒子 |
