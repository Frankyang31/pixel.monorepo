# AI 内容生成平台 — 网站页面清单

> 整理于 2026-03-24，基于架构设计方案与产品讨论梳理而成。
> 本文档作为后续 UI 设计、前端开发的页面基准，随迭代持续更新。

---

## 基本信息

| 项目 | 说明 |
|------|------|
| 目标市场 | 国内 + 海外（中英文双语） |
| 前端框架 | Vue 3 + Nuxt 3（SSR/SSG 混合渲染） |
| 渲染策略 | 公域页 SSG/SSR（SEO 优先），产品页 CSR（交互优先） |
| 总页面数 | 22 个 |

---

## 一、公域获客层（9 页）

> SEO 核心战场，采用 SSG 或 SSR 渲染，确保搜索引擎可收录。

| # | 页面名称 | 路由 | 渲染策略 | 核心目标 |
|---|---------|------|---------|---------|
| 1 | 首页 | `/` | SSG + ISR | 品牌第一印象 + 工具入口 + 精选案例展示 |
| 2 | 工具列表页 | `/tools` | SSG | 所有场景工具的导航聚合，SEO 中枢页 |
| 3 | 工具详情页 | `/tools/[slug]` | SSR | 每个场景独立 URL，长尾关键词 SEO 核心，含工具介绍、使用教程、案例、FAQ |
| 4 | 作品广场 | `/gallery` | SSR | 用户公开作品瀑布流，贡献内容 SEO，混排（后续可调整排序算法） |
| 5 | 作品详情页 | `/gallery/[id]` | SSR | 单个作品独立页，可被搜索引擎收录，含作品信息、使用的工具/参数、作者信息 |
| 6 | 博客列表页 | `/blog` | SSG | 教程、案例、行业内容聚合 |
| 7 | 博客详情页 | `/blog/[slug]` | SSG + ISR | Markdown 渲染，长尾关键词覆盖 |
| 8 | 定价页 | `/pricing` | SSG | 会员等级对比（L0–L3）+ 购买引导，变现转化关键页 |
| 9 | 活动落地页 | `/activity/[slug]` | SSG | 节日/合作/限时活动专题，预渲染最优性能 |

## 二、核心产品层（2 页）

> 用户登录后的主要使用区域，采用 CSR 渲染，优先交互体验。

| # | 页面名称 | 路由 | 说明 |
|---|---------|------|------|
| 10 | 工作台 | `/studio` | 平台核心页。布局：**左侧表单区**（选择场景/模型、填写 Prompt、配置参数）+ **右侧结果区**（生成中任务占位卡片、生成结果展示、支持多任务并行）。任务进度通过 SSE 实时推送，内嵌在右侧结果区，不跳转独立页。文生图 / 图生视频等不同模式在同一页内切换。 |
| 11 | 我的作品集 | `/studio/history` | 个人历史生成记录列表，支持筛选（按工具/时间/状态），可下载、删除、设置为公开/私密。 |

## 三、账户与会员层（5 页）

> 用户个人信息与变现路径，CSR 渲染。

| # | 页面名称 | 路由 | 说明 |
|---|---------|------|------|
| 12 | 个人中心 | `/account` | 头像、昵称、邮箱/手机绑定、账号安全设置、第三方账号绑定 |
| 13 | 积分明细 | `/account/points` | 积分获取/消耗流水记录，含预扣/冻结状态展示，支持分页查询 |
| 14 | 会员管理 | `/account/membership` | 当前会员等级与权益展示、到期时间、续费/升级入口 |
| 15 | 支付页 | `/checkout` | 选择会员套餐或积分包 → 选择支付方式（支付宝/微信/Stripe）→ 确认下单 |
| 16 | 支付结果页 | `/checkout/result` | 支付成功/失败反馈，积分到账确认，引导回工作台 |

## 设计优先级建议

按商业价值和开发依赖关系，建议以下顺序推进设计：

| 优先级 | 页面 | 理由 |
|--------|------|------|
| P0 | 首页、工作台 | 品牌第一印象 + 核心体验，一切的基础 |
| P0 | 登录/注册页 | 用户进入产品的门槛，必须先完成 |
| P1 | 定价页、支付页 | 变现转化路径，需尽早上线 |
| P1 | 工具详情页 | SEO 流量承接，影响增长曲线 |
| P2 | 作品广场、作品详情 | 内容 SEO 引擎，需有一定存量作品后再发力 |
| P2 | 个人中心、积分明细 | 用户管理，不影响核心转化 |
| P3 | 博客体系、活动落地页 | 持续运营内容，后续补充 |

## 关键设计决策记录

| 决策点 | 结论 | 备注 |
|--------|------|------|
| 工作台布局 | 左表单 + 右结果，单页切换模式 | 参考 Leonardo.ai、Midjourney Web 布局范式 |
| 任务进度展示 | 内嵌右侧结果区，不跳独立页 | 支持多任务并行占位展示 |
| 作品公开机制 | 用户可选择是否公开作品 | 默认私密，用户主动设为公开后出现在广场 |
| 作品广场排序 | 初期混排 | 后续根据数据迭代排序算法 |
| 国际化 | 中英文双语 | 路由层面支持 i18n，SEO 元数据双语配置 |
| 认证方式 | 微信 OAuth（国内）+ Google OAuth（海外）+ 邮箱/手机 | 兼顾两个市场的登录习惯 |
