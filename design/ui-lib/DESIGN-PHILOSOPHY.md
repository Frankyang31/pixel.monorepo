# 组件库设计理念 — "深夜工作室"

> 我们不是在做一个 SaaS 后台管理系统，而是在做一个**创作者的沉浸式工具**。
> 组件的终极目标：让界面安静退到背景，让用户的作品成为唯一的主角。

---

## 一、核心设计语言

### 1.1 氛围感优先

不是"好看"，而是"有氛围"。组件不能只是"功能正确"，它要传递一种情绪——
当你打开工作台，感觉像是走进了一间灯光柔和的深夜工作室。

**具体手法：**

- **发光替代投影** — 深色主题下所有"层次感"用发光（glow）表达，而非传统 drop-shadow。
  阴影是"遮挡"，发光是"光从物体背后溢出"——完全不同的物理隐喻。
- **玻璃质感（Glassmorphism）** — 卡片、弹窗、浮层面板使用 `backdrop-filter: blur()`，
  让底层内容隐约可见，营造"透明材质叠在画布上"的层次感。
- **渐变即品牌** — 渐变不只是按钮的颜色，它是整个设计系统的"光"。指示器、选中态、
  进度条、边框高亮……凡是需要"活跃感"的地方，都应该有渐变的参与。

### 1.2 交互即反馈

创作者在等待 AI 生成时，每一秒都漫长。组件的微交互不是为了炫技，
而是为了让用户**感知到系统是活的**。

**具体手法：**

- **聚焦光晕** — 输入框聚焦时，品牌色从边框向外扩散为柔和光晕（`box-shadow` + `border-color`），
  而不是简单的描边变色。深色主题下效果尤其明显。
- **弹性按压** — 按钮 active 时不是简单 `translateY(1px)`，而是配合 `scale` 的弹性回弹，
  让点击有一种"按压弹簧"的触感。用 `ease-spring` 缓动实现。
- **渐变边框旋转** — Loading / 生成进行中状态，用 `conic-gradient` + `@keyframes rotate`
  实现边框上的渐变旋转，替代传统 spinner。这是这个产品独有的视觉签名。
- **状态过渡编排** — 所有状态切换（hover → active → focus → disabled）都经过时序编排，
  避免所有属性同时变化导致的"闪烁感"。背景先行，然后阴影，最后内容颜色。

### 1.3 克制的力量

氛围感不代表堆砌。该克制的地方必须克制：

- **留白是呼吸** — 组件内间距偏宽（参考设计规范的 `gap-md` 16px），不塞满。
- **颜色做减法** — 深色主题下大量使用透明度层级（`rgba(255,255,255,0.06)` 等），
  而非新增颜色。让渐变品牌色成为视觉焦点，其余组件尽量安静。
- **文字层级清晰** — 严格遵守 `text-primary > text-secondary > text-tertiary` 三级层级，
  不在同一组件内出现 4 种以上文字颜色。

## 二、深色主题 vs 浅色主题差异化策略

| 维度 | 深色主题（主战场） | 浅色主题（营销场景） |
|------|-------------------|---------------------|
| 层次感 | 发光 + 玻璃模糊 | 传统投影 + 微妙背景色差 |
| 边框 | 极低透明度白色线 + 品牌色发光 | 低透明度黑色线 + 品牌色描边 |
| Hover | 玻璃态高亮 + 柔和发光 | 背景提亮 + 阴影加深 |
| Focus | 品牌色光晕（品牌色 box-shadow） | 品牌色描边 + 浅色光晕 |
| 背景 | `#0D0D12` 带微紫，营造"暗房感" | `#F5F4F8` 暖灰，营造"画纸感" |
| Loading | 渐变边框旋转 + 脉冲光晕 | 渐变进度条 + 淡入淡出 |

## 三、核心组件差异化设计方向

### 3.1 Button 按钮 — "发光的行动召唤"

| 要素 | 设计 |
|------|------|
| Primary | 渐变底色不变，但 hover 时光晕从按钮底部溢出（`box-shadow: 0 4px 20px rgba(123,94,240,0.4)`），active 时弹性回弹 `scale(0.97)` |
| Secondary | 深色下边框使用 `rgba(255,255,255,0.10)`，hover 时边框渐变为品牌色，触发玻璃态背景 `rgba(123,94,240,0.06) + backdrop-blur` |
| Ghost | hover 时背景是极淡的玻璃态 `rgba(255,255,255,0.04)` 而非实色 |
| Loading | **渐变边框旋转** — 用 `conic-gradient(from var(--angle), #7B5EF0, #4F8EF7, transparent)` 做 border-image，配合 CSS Houdini `@property --angle` 实现旋转动画 |
| Progress | 新增 `btn--progress` 变体 — 按钮底部有一条渐变进度条（类似 YouTube 加载条），从左到右填充 |

### 3.2 Input 输入框 — "安静的画框"

| 要素 | 设计 |
|------|------|
| 背景 | 深色下用 `rgba(255,255,255,0.04)` + `backdrop-blur(4px)` 的极淡玻璃态，而非实色 `#252535` |
| Focus | 品牌色光晕从边框向外扩散 — `box-shadow: 0 0 0 1px var(--color-brand-mid), 0 0 20px rgba(123,94,240,0.15)` |
| Label | 浮动标签（floating label）：placeholder 在上方显示，输入时 label 上浮缩小，减少视觉噪音 |
| Prompt 模式 | 新增 `input--prompt` — mono 字体，更大行高，支持多行自动扩展，左侧有一条极淡的渐变色竖线作为"画布边界"暗示 |
| Search | 搜索图标在左侧，右侧清除按钮 hover 时带渐变圆形背景 |

### 3.3 Card 卡片 — "漂浮的画框"

| 要素 | 设计 |
|------|------|
| 常规态 | 深色下边框 `rgba(255,255,255,0.06)`（设计规范的 `border-subtle`），而非 `border-default` |
| Hover | 边框亮度提升至 `rgba(255,255,255,0.12)`，同时触发极淡的品牌色发光 `box-shadow: 0 0 30px rgba(123,94,240,0.08)` |
| 选中态 | 品牌色渐变边框（通过 `border-image` 或 `background-clip` 技巧实现渐变描边），配合外发光 |
| 画廊模式 | 新增 `card--gallery` — 去掉所有文字信息，纯图片展示，hover 时底部渐变蒙版上浮显示标题 |
| Stats 卡片 | 数字使用品牌渐变色（`background-clip: text`），hover 时卡片有微弱上浮 + 发光 |

### 3.4 Tabs 选项卡 — "安静的导航灯"

| 要素 | 设计 |
|------|------|
| Underline | 指示器不是 2px 实线，而是 2px **渐变色**（`background: linear-gradient(90deg, #7B5EF0, #4F8EF7)`），活跃 Tab 的文字也带渐变色（`background-clip: text`） |
| Pill | 容器用玻璃态背景，活跃项用渐变填充，同时带有极淡的外发光 |
| Segment | 容器深色下用玻璃态，活跃项带品牌色渐变边框（而非实色背景） |
| 切换动画 | 指示器用 `transform: translateX()` 滑动而非即时切换，时序 `duration-slow + ease-out` |

### 3.5 Badge 徽章 — "发光的标签"

| 要素 | 设计 |
|------|------|
| 品牌 Badge | 渐变底色不变，新增**外发光** `box-shadow: 0 0 10px rgba(123,94,240,0.3)` |
| 功能 Badge | 成功/警告/错误使用**玻璃态底色** `rgba(color, 0.12)` + `backdrop-blur`，而非实色背景 |
| 计数 Badge | 小圆点脉冲动画 — 新消息时圆点有 `pulse` 呼吸灯效果 |
| 渐变文字 | "New" / "Pro" 等强调标签，文字本身使用品牌渐变色 |

### 3.6 Modal 弹窗 — "画布上的浮窗"

| 要素 | 设计 |
|------|------|
| 遮罩 | `rgba(0,0,0,0.6)` + `backdrop-filter: blur(8px)`，底层内容完全模糊化 |
| 面板 | 玻璃态背景 `rgba(19,19,26,0.85)` + `backdrop-blur(20px)` + `border: 1px solid rgba(255,255,255,0.10)` |
| 边框 | 顶部边缘有一条极淡的品牌色渐变光线（模拟"光从窗口上方洒入"） |
| 动画 | 进入时从 `opacity: 0 + scale(0.95) + translateY(10px)` 到正常状态，`duration-slow + ease-out` |
| 关闭按钮 | hover 时带渐变圆形背景（类似 macOS traffic light 的柔和感） |

### 3.7 Toast 通知 — "滑入的光条"

| 要素 | 设计 |
|------|------|
| 容器 | 玻璃态背景 + 左侧 3px 品牌渐变色条（作为状态指示） |
| 成功态 | 左侧条为 `--color-accent-teal`，带柔和青绿发光 |
| 错误态 | 左侧条为 `--color-accent-red`，带柔和红色发光 |
| 进入动画 | 从右侧 `translateX(100%)` 滑入 + `opacity 0→1`，`duration-slow + ease-out` |
| 退出动画 | 向右滑出 + `opacity 1→0`，`duration-normal + ease-in` |

## 四、不做的清单

以下效果**明确不使用**，避免过度设计：

1. **粒子效果** — 纯装饰性粒子会增加 DOM 负担，且与"界面退到背景"理念冲突
2. **彩虹渐变** — 所有渐变严格限定在品牌紫蓝色域内
3. **3D 变换** — 不使用 `perspective` / `rotate3d`，保持界面平面感
4. **霓虹文字发光** — `text-shadow` 的发光效果对比度差，影响可访问性
5. **过度模糊** — `backdrop-blur` 上限 20px，避免完全遮盖底层内容导致信息丢失

## 五、可访问性底线

无论视觉上做多少差异化，以下规则不可破坏：

- 文字对比度 ≥ 4.5:1（WCAG AA）
- 焦点指示器始终可见（`focus-visible`）
- 所有交互元素支持键盘操作
- `prefers-reduced-motion` 下关闭所有非必要动画
- 渐变文字必须提供 fallback 颜色（`color: var(--color-brand-mid)` 作为 fallback）
