# 工作台设计说明

**原型文件**：`studio-design.html`  
**主题**：Dark Theme（深色产品主题）  
**渲染策略**：CSR（客户端渲染，交互优先）  
**路由**：`/studio`  
**最后更新**：2026-03-24  

---

## 设计目标

工作台是整个产品的核心页面，用户在此完成「选场景 → 填 Prompt → 配置参数 → 生成 → 查看结果」的完整创作流程。

设计需要解决的核心问题：

1. **降低创作门槛**：即使是第一次使用的用户，也能在 30 秒内完成第一次生成
2. **支持多任务并行**：用户提交生成后可以立刻再次提交，结果区同时显示多个任务状态
3. **最小化等待焦虑**：AI 生成耗时长，需要让用户清楚感知进度，而不是盯着黑屏等待
4. **功能密度与清洁感的平衡**：参数很多，但不能让界面显得拥挤复杂

---

## 主题策略

工作台采用 **Dark Theme（深色主题）**，理由：

- 创作工作台需要高度专注，深色界面减少环境光对内容的干扰
- 用户在工作台停留时间长（10-30 分钟），深色对眼睛更友好
- 生成的图片/视频在深色背景下展示效果更好，色彩更饱和
- 与首页浅色形成主题切换，强化「进入产品工作区」的沉浸感

背景底色 `#0D0D12`（极深紫黑）——比纯黑 `#000000` 多一点微紫调，呼应品牌色，有自己的调性。

## 布局结构

```
┌─────────────────────────────────────────────────────────┐
│ Top Nav Bar（48px，固定顶部）                           │
├──────────────────────────┬──────────────────────────────┤
│                          │                              │
│   Left Panel（340px）    │   Right Result Area（flex）  │
│   左侧控制面板           │   右侧结果区                 │
│                          │                              │
│   固定不滚动             │   可滚动                     │
│                          │                              │
└──────────────────────────┴──────────────────────────────┘
```

**关键布局决策**：左侧面板固定宽度 340px、不随内容滚动，右侧结果区 flex-1、独立滚动。理由：用户调整参数时不应影响结果区视图；查看结果时左侧参数随时可操作。

## 区块结构与设计决策

### 1. Top Nav Bar（顶部导航栏）

**内容**：Logo（左）+ 模式切换 Tab（中）+ 积分余额 + 头像（右）

**设计决策**：
- 模式切换放在顶部而非左侧：模式（文生图/图生视频/风格迁移）是页面级切换，影响整个布局，顶部位置更符合用户认知模型（参考 Figma、Sketch 等创作工具）
- 积分余额始终可见，避免用户在提交生成时才意识到积分不足
- 导航高度 48px，比标准 64px 更紧凑，给创作区域让出更多空间

**与设计规范关联**：`--bg-surface`（`#13131A`）作为 Nav 背景，区别于页面底色

---

### 2. Left Panel — 模式切换（顶部 Tab）

**内容**：文生图 / 图生视频 / 风格迁移 / 更多（下拉）

**设计决策**：
- 激活状态用品牌色底部下划线 + 浅色背景，而非改变文字颜色——下划线指示器在多 Tab 场景下视觉权重更合适
- 「更多」Tab 保留扩展入口，不强迫在此版本就列出所有工具

---

### 3. Left Panel — 模型选择器

**内容**：当前选中模型名 + 风格色块 + 下拉切换

**设计决策**：
- 每个模型用一个独特的渐变色块作为视觉标识，让用户快速识别当前选中的是哪个模型，不依赖纯文字
- 模型选择影响后续所有参数的可用状态，放在面板顶部突出位置

---

### 4. Left Panel — Prompt 输入区

**内容**：主 Prompt 文本框 + 字数计数 + 工具按钮（优化/翻译/随机）

**设计决策**：
- 文本框使用等宽字体 `JetBrains Mono`，让 Prompt 编写有「精确输入」的仪式感，区别于普通表单
- 聚焦时出现品牌色光晕边框（`box-shadow: 0 0 0 3px rgba(123,94,240,0.2)`），强化「创作焦点在这里」的感知
- 字数计数实时更新，防止超出模型限制
- 三个快捷工具按钮：「优化」（AI 润色 Prompt）、「翻译」（中文转英文）、「随机」（随机生成灵感）—— 降低 Prompt 编写门槛

---

### 5. Left Panel — 反向提示词（Negative Prompt）

**内容**：折叠式输入区，默认收起

**设计决策**：
- 默认折叠：反向提示词是进阶功能，对新用户可能造成困惑，折叠起来降低初次认知负荷
- 点击标题展开，展开时显示带动画，体验流畅

---

### 6. Left Panel — 比例选择器

**内容**：5 档比例选项（1:1 / 4:3 / 16:9 / 3:4 / 9:16）

**设计决策**：
- 使用可视化小方块而非文字下拉，让用户「看到」比例而不是「读到」比例
- 每个比例方块的长宽比与实际选项一致（如 16:9 的选项就是宽扁矩形）
- 当前选中态：品牌色边框 + 半透明品牌色背景

---

### 7. Left Panel — 高级参数

**内容**：折叠块，含质量/创意程度/推理步数滑块 + 随机种子输入

**设计决策**：
- 默认折叠：同反向提示词，高级参数面向进阶用户，不应干扰主流程
- 滑块使用自定义样式，品牌色轨道，确保深色背景下可见性
- 随机种子支持手动输入，满足专业用户「复现某次结果」的需求

---

### 8. Left Panel — 生成数量 & 底部操作栏

**内容**：生成数量选择（1/2/4）+ 积分消耗预览 + 生成按钮

**设计决策**：
- 底部固定不随左侧内容滚动，生成按钮始终可见
- 积分消耗实时计算（数量×单次消耗），让用户在点击前知道会扣多少分
- 生成按钮使用品牌渐变色，是整个左侧面板视觉权重最高的元素
- 支持 `⌘ + ↵` 快捷键提交，面向习惯键盘操作的用户

---

### 9. Right Area — 工具栏

**内容**：状态筛选 Tab（全部/生成中/已完成/已收藏）+ 布局切换（网格/列表）

**设计决策**：
- 筛选放在结果区顶部而非左侧，符合用户「先提交，再看结果」的操作顺序
- 布局切换：网格适合浏览多张图，列表适合查看详细信息（后续迭代）

---

### 10. Right Area — 任务卡片（生成中）

**内容**：模糊的图片占位背景 + SVG 进度环动画 + 百分比 + 任务描述

**设计决策**：
- 进度环而非进度条：圆形进度环在正方形卡片中视觉更均衡，参考 iOS 下载指示器
- 进度环实时动画（前端模拟），即使后端 SSE 推送有延迟，视觉上也给用户「在进行中」的持续反馈
- 背景使用模糊渐变，让「等待状态」有美感而不是空白
- 卡片大小与完成卡片保持一致，切换时布局不跳动

**实际数据对接**：进度百分比来自 SSE 实时推送，前端只负责显示；原型中用 JS 定时器模拟

---

### 11. Right Area — 结果卡片（已完成）

**内容**：生成图片 + hover 操作按钮（收藏/下载/更多）+ 底部元数据

**设计决策**：
- hover 时操作按钮浮现（而非常驻），保持卡片干净，让图片内容本身成为主角
- 底部元数据：模型名 + 公开/私密状态指示——用户需要快速知道这张图的属性
- 「公开/私密」状态用眼睛图标直觉化表示

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 模式 Tab 切换 | 左侧面板内容切换（保留 Prompt） | Prompt 在模式间共享，降低重复输入 |
| 比例选择 | 即时切换高亮 | 无需确认 |
| 参数折叠块 | 点击标题展开/收起，带滑动动画 | `max-height` 过渡实现 |
| 生成数量选择 | 底部积分数字实时更新 | 数量×单次消耗动态计算 |
| 提交生成 | 结果区顶部插入「生成中」卡片 | 不清空 Prompt，支持连续提交 |
| 进度推进 | SSE 推送 → 进度环更新 → 完成后替换为结果卡片 | 原型中用定时器模拟 |
| 键盘快捷键 | `⌘ + ↵` 触发生成 | 面向效率用户 |

## 响应式策略

工作台为「桌面优先」设计，移动端体验降级：

| 断点 | 布局变化 |
|------|----------|
| ≥1024px | 左右分栏，完整功能 |
| 768-1023px | 左侧面板收窄至 280px |
| <768px | 左侧面板隐藏，底部抽屉展开参数（后续迭代） |

---

## 可访问性说明

- 所有表单元素有 `<label>` 关联（`aria-label` 或显式 label）
- 进度环有 `aria-valuenow` / `aria-valuemin` / `aria-valuemax` 属性
- 折叠块使用 `aria-expanded` 状态
- 滑块使用原生 `<input type="range">`，天然支持键盘操作
- 生成按钮在提交中状态有 `disabled` + `aria-busy` 状态
- 色彩对比度：主文字 `#F0F0F8` on `#0D0D12` ≈ 18:1
