# 首页设计说明

**原型文件**：`homepage-design.html`  
**主题**：Light Theme（浅色营销主题）  
**渲染策略**：SSG（静态生成，SEO 优先）  
**最后更新**：2026-03-24  

---

## 设计目标

首页是产品对外的第一张名片，承担三个核心任务：

1. **建立品牌认知**：在 3 秒内让用户明白「这是一个什么产品」
2. **激发创作欲望**：通过作品预览和工具展示，让用户产生「我也想试试」的冲动
3. **引导转化注册**：降低决策门槛（免费起步），推动用户点击注册

目标用户：对 AI 生成内容感兴趣但尚未注册的新用户。核心转化路径：**进入首页 → 被 Hero 吸引 → 看到作品案例 → 点击「免费开始」注册**。

---

## 主题策略

首页采用 **Light Theme（浅色主题）**，理由：

- 营销页面需要建立信任感，浅色系更接近用户日常浏览习惯（参考 Notion、Linear 等标杆产品）
- 深色主题留给产品内部（工作台），形成「进入产品后沉浸感增强」的体验落差
- 对 SEO 和无障碍审查更友好

背景底色选用 `#F5F4F8`（暖灰，非纯白），比纯白多一点温度，呼应品牌的「创意工作室感」。

**Hero 区块采用浅色背景 + 品牌色柔光光晕**（而非深色全屏），与页面整体浅色调保持一致，同时通过 CSS 动画光晕和渐变文字保持视觉冲击力。工具展示区、作品画廊等区块均使用白色/暖灰背景，形成连贯的浅色体验。CTA Banner 使用品牌渐变色作为视觉高潮，与整体浅色基调形成对比。

---

## 区块结构与设计决策

### 1. Nav（导航栏）

**内容**：Logo（左）+ 主导航链接（中）+ 登录/注册按钮（右）

**设计决策**：
- 采用 `position: sticky` 吸顶，滚动时触发毛玻璃模糊背景（`backdrop-filter: blur(20px)`），避免滚动过程中导航遮挡内容显得突兀
- 注册按钮用品牌渐变色，登录按钮用透明底，视觉权重区分明确
- 导航链接：工具、广场、定价、博客 —— 对应用户最关注的四个决策点

**与设计规范关联**：`--gradient-brand`、`--r-full`（圆形按钮）、`--t-normal`（hover 动效）

---

### 2. Hero（主视觉区）

**内容**：大标题 + 副标题 + 主/次 CTA 按钮 + 品牌光晕背景 + 工作台预览模拟图

**设计决策**：
- 标题采用渐变文字（`background-clip: text`），强化品牌色印象，比纯色文字更有视觉冲击力
- 背景采用多个模糊光晕（radial-gradient），而非插图或视频，原因：技术实现简单、加载快、不分散注意力
- Hero 区内嵌一个工作台 UI 预览（深色模拟图），直接展示产品界面，用户无需想象「这个工具长什么样」——这是对比 Leonardo.ai（纯文字+图片）的差异化做法
- CTA 主按钮文案「免费开始创作」强调免费降低门槛，次按钮「浏览作品广场」提供另一条无压力进入路径

**与设计规范关联**：`--gradient-brand`、`--shadow-brand`、`--bg-overlay`（预览卡片）

---

### 3. Social Proof（数据信任区）

**内容**：四项核心数据指标 + 描述文字

**指标选择**：
- 注册创作者数量（用户规模）
- 生成作品数量（活跃度）
- 平均用户评分（质量口碑）
- 支持 AI 模型数量（技术能力）

**设计决策**：
- 数字用品牌渐变色，视觉上构成「品牌色高光节拍」，强化数字的重要性
- 四列等分网格，移动端折叠为 2×2，保持阅读密度

---

### 4. Tools Showcase（工具展示区）

**内容**：6 个工具卡片，每张含工具名、场景描述、标签、积分消耗、进入按钮

**当前工具列表**：文生图、图生视频、风格迁移、超分辨率、背景消除、AI 智能修图

**设计决策**：
- 每个工具卡片左上角有品牌渐变色 icon，统一视觉语言
- 卡片底部显示「从 X 积分」—— 让用户在首页就了解消耗预期，减少进入工作台后的「惊吓感」
- hover 时卡片上浮 + 出现品牌色光晕边框，增强交互反馈
- 工具数量 6 个：3×2 网格布局，既不太少显得产品单薄，也不太多增加认知负荷

**待更新**：工具名称和数量以实际产品规划为准，此处为占位设计。

---

### 5. Gallery Preview（作品画廊预览）

**内容**：精选 AI 生成作品的瀑布流展示 + 进入广场按钮

**设计决策**：
- 作品展示是最强的「所见即所得」信任建立方式——用户看到效果图比任何文案都有说服力
- 使用 CSS multi-column 瀑布流，hover 时卡片放大并显示作品标签和作者
- 当前使用 CSS 渐变色块作为作品占位，上线后替换为真实生成图片
- 底部「查看更多」引导进入作品广场，属于第二条转化路径（对不急于注册的用户）

---

### 6. Pricing Teaser（定价预告区）

**内容**：免费/Pro/Max 三档会员卡片

**设计决策**：
- 首页只做「预告」而不是完整定价页，目的是消除「这个工具要花多少钱」的疑虑
- Pro 档视觉高亮（品牌渐变卡片 + 「最受欢迎」标签），引导用户关注最有价值的档位
- 每档只展示最核心的 3 个权益，不做完整功能对比——完整对比在定价页

---

### 7. CTA Banner（注册引导横幅）

**内容**：全宽渐变横幅 + 大号标题 + 注册按钮 + 信任文案

**设计决策**：
- 放在 Footer 前的最后一个区块，承接完整浏览后的「准备好注册了」心理状态
- 文案「无需信用卡，立即免费开始」直接解除两个常见顾虑（付费和注册复杂度）
- 横幅背景使用品牌渐变色，是整个浅色页面中视觉最强烈的区块，起到「终章」的收尾感

---

### 8. Footer

**内容**：四列链接 + 社交媒体图标 + 版权文字

**设计决策**：
- Footer 使用深色背景（`#18181F`），与页面浅色主体形成强对比，形成自然的视觉「落地感」
- 社交媒体图标列表：Twitter/X、Discord、GitHub、微信 —— 覆盖国内外主要渠道

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥1280px | 最大容器宽度，完整布局 |
| 1024-1279px | 容器收窄，工具卡片保持 3 列 |
| 768-1023px | 工具卡片变 2 列，Hero 缩小字号 |
| <768px | 单列布局，工具卡片 1 列，导航折叠，数据指标 2×2 |

---

## 可访问性说明

- 所有交互元素有 `:focus-visible` 状态（品牌色轮廓）
- 色彩对比度：主文字 `#18181F` on `#F5F4F8` ≈ 16:1（远超 WCAG AA 要求的 4.5:1）
- 支持 `prefers-reduced-motion`：动画效果在此偏好下禁用
- 语义化 HTML 结构：`<nav>`、`<main>`、`<section>`、`<footer>` 正确使用
- 图片/icon 有 `alt` 属性占位
