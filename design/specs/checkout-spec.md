# 支付页 & 支付结果页 设计说明

**原型文件**：`checkout-design.html`  
**主题**：Dark Theme  
**渲染策略**：CSR（需登录态、实时订单数据）  
**路由**：`/checkout`  
**最后更新**：2026-03-24

---

## 设计目标

支付页是用户完成消费转化的最后一步，核心目标是**降低决策摩擦、建立支付信任感、减少中途放弃**。目标用户是已选定套餐或积分包准备付款的用户。核心路径：确认订单 → 选择支付方式 → 完成支付 → 进入成功页 → 回到工作台。

---

## 主题策略

采用 Dark Theme，与产品内页保持一致（工作台、账户系列均为暗色）。暗色背景在支付场景有助于聚焦用户注意力，减少视觉干扰，同时营造"专业感"和"安全感"。

---

## 区块结构与设计决策

### 1. 顶部导航（极简版）

**内容**：仅保留 Logo + 安全锁标识，无页面导航链接。

**设计决策**：
- 支付流程中去掉导航链接，防止用户分心跳出——这是电商支付页的标准做法（参考支付宝、微信支付结账页）。
- 右侧"安全支付"标识（🔒）降低用户对安全性的疑虑。

**与设计规范关联**：`--bg-base`、`--border-subtle`、`--font-sans`

---

### 2. 双栏布局（左订单 + 右支付方式）

**内容**：左栏为订单信息确认，右栏为支付方式选择和付款按钮。

**设计决策**：
- 双栏保持左右信息对称，用户可在确认金额的同时选择支付方式，减少来回滚动。
- 左栏宽度略大于右栏（约 1.2:1），符合"主要信息优先"的阅读习惯。
- 移动端折叠为单列，顺序为：订单概要 → 支付方式 → 付款按钮。

**与设计规范关联**：`--bg-surface`、`--radius-xl`、`--shadow-md`

---

### 3. 订单信息卡（左栏）

**内容**：套餐名称、套餐特权列表（checkmark 样式）、价格明细（原价/折扣/实付）、有效期说明。

**设计决策**：
- 权益列表帮助用户在最后一步再次确认购买价值，减少"付了钱不知道买了啥"的焦虑。
- 价格明细分行展示，划线原价+高亮实付价，强化优惠感。
- 会员 Badge（Pro/Free 样式）提升套餐视觉感知价值。

**与设计规范关联**：`--brand-start`、`--accent-amber`、`--bg-elevated`

---

### 4. 支付方式选择（右栏）

**内容**：微信支付、支付宝、信用卡三种选项，Radio 单选样式，选中态有品牌色高亮边框。

**设计决策**：
- 优先展示国内用户最常用的支付方式（微信 > 支付宝 > 卡支付），符合国内用户习惯。
- 信用卡选项展开后显示卡号输入表单（CSS 折叠动画），避免表单常驻占用空间。
- 选中态使用品牌色边框（`--border-brand`）而非填充色，避免影响内容可读性。

**与设计规范关联**：`--border-brand`、`--brand-start`、`--radius-xl`

---

### 5. 付款按钮与法律声明

**内容**：大号渐变"立即支付"按钮 + 下方小字服务协议/隐私政策链接。

**设计决策**：
- 按钮使用全宽样式（在右栏容器内），视觉重量最大，引导用户点击。
- 协议确认采用默认勾选+小字说明方式，符合国内平台惯例，减少额外操作步骤。

**与设计规范关联**：`--brand-start`、`--brand-end`、`--font-sans`

---

### 6. 支付成功页（同文件内状态切换）

**内容**：全屏居中祝贺动画（彩纸）+ 套餐激活确认 + 积分到账说明 + 跳转工作台按钮。

**设计决策**：
- 支付成功与支付页同文件，通过 CSS class 切换（`checkoutPage` hide / `successPage` show），避免跳转带来的加载等待感。
- 彩纸动画（canvas confetti）增强仪式感和成就感，有助于提升用户满意度和复购意愿。
- 成功页主要 CTA 为"开始创作"，引导用户立即回到工作台消费积分，形成使用闭环。

**与设计规范关联**：`--accent-teal`、`--accent-amber`

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 点击支付方式 Radio | 高亮边框，信用卡选项展开卡号表单 | CSS transition 动画 |
| 点击"立即支付" | 触发支付动效，成功后切换到成功页 | JS class 切换 |
| 成功页彩纸动画 | 页面切换后自动触发 canvas 动画 | createConfetti() 函数 |
| 点击"开始创作" | 跳转到 /studio（工作台） | 页面跳转 |
| 点击"查看订单" | 跳转到订单详情 | 页面跳转 |

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥1280px | 双栏（左订单 + 右支付），侧边各留白 |
| 1024-1279px | 双栏保持，卡片内边距略减 |
| 768-1023px | 单列，顺序：套餐卡 → 支付方式 → 按钮 |
| <768px | 单列全宽，底部固定"付款"按钮栏 |

---

## 可访问性说明

- 支付方式 Radio 使用原生 `<input type="radio">` 配合 `<label>`，确保键盘可操作。
- 付款按钮 `aria-label="立即支付 ¥298"` 包含金额，避免屏幕阅读器读取不完整。
- 成功页动画提供静态降级版本（prefers-reduced-motion 媒体查询）。
- 文字与背景对比度：价格白字 on `--bg-surface` ≥ 7:1（AAA）。
