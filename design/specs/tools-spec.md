# 工具列表页 设计说明

**原型文件**：`tools-design.html`
**主题**：Light Theme
**渲染策略**：SSG（工具列表相对稳定，SEO 友好）
**路由**：`/tools`
**最后更新**：2026-03-25

---

## 设计目标

工具列表页是游戏开发者发现和选择 AI 资产生成工具的入口，核心目标是**帮助游戏开发者快速找到适合其项目阶段的工具并进入使用**。目标用户有两类：①首次来访的独立游戏开发者（通过搜索引擎或首页跳转），②已有项目需要特定资产类型的老用户。核心路径：浏览工具 → 点击工具卡片 → 进入工具详情页 → 进入工作台。

---

## 主题策略

采用 Light Theme，与首页、定价页保持一致，属于"公域获客层"页面。浅色背景在工具卡片展示时能更好地衬托每张卡片独特的渐变色缩略图，增强视觉吸引力。同时有利于 SEO 内容抓取和社交分享预览。

## 区块结构与设计决策

### 1. 顶部页头（Hero）

**内容**：页面标签（"🎮 游戏资产生成平台"）+ 大标题 + 副标题说明 + 双 CTA + 工具统计数字。

**设计决策**：
- Hero 文案从泛化的"AI 创作工具"改为"游戏美术资产"，标题明确传递游戏垂类定位
- 双 CTA："立即免费试用"（主）+ "查看全部工具"（次），引导用户快速行动
- 底部统计数字反映真实游戏开发者数据（10K+ 开发者），而非泛化的总用户数

**与设计规范关联**：`--brand-start`、`--brand-end`、`--bg-base`

---

### 2. 工具 Intro Banner（新增，2026-03-25）

**内容**：深色背景（`#1a1a2e`）的区块，左侧产品价值主张文案，右侧三个数据（80% 工时节省 / ~6s 生成速度 / PNG/WebP 格式）。

**设计决策**：
- 深色背景与浅色页面形成对比，强调这是"专业创作工具区"
- 数据驱动的文案（"80% 工时节省"）直接回应开发者"AI 能帮我省多少时间"的疑问
- 定位在 Hero 之后、工具网格之前，是工具列表的"序曲"

---

### 3. 分类标签过滤栏

**内容**：横向 Tab 标签（全部 / 角色与立绘 / 场景与资产 / 量产与精修 / 视频与动效），支持横向滚动。

**分类设计决策**：
- 按"游戏制作工作流阶段"而非"工具技术类型"分类，更贴近用户思维模型
- "角色与立绘"只有一个工具但单独成类，强调其核心地位
- Tab 横向可滚动，移动端体验一致

**与设计规范关联**：`--radius-full`、`--border-brand`、`--bg-elevated`

---

### 4. 工具卡片网格（核心区块）

**布局**：3 列 × 2 行，共 6 张工具卡。

**每张卡片设计**：
- 缩略图：每个工具使用独特的渐变背景色，匹配工具属性的视觉隐喻
  - 角色立绘：暗紫+金色光 → 科技感、质感
  - 场景原画：深蓝+青光 → 自然感、氛围感
  - 动画预览：暗橙+珊瑚光 → 动态感、能量感
  - Sprite 批量：深绿调 → 像素感、复古感
  - 材质纹理：蓝灰工业风 → 材质感、精确感
  - 细节精修：暖粉精确 → 精细感、精确感
- 缩略图内含 emoji 图标 + 右上角 Badge（热门/Pro/新功能）
- 卡片悬浮升起效果（`translateY(-4px)`），视觉反馈更强
- 积分消耗标识使用紫色品牌色（`--brand-mid`），统一但不抢眼
- 每个工具关联游戏类型标签（RPG、SLG、二次元等），强化垂类感知

**与设计规范关联**：`--bg-surface`、`--shadow-md`、`--radius-2xl`、`--brand-mid`

---

### 5. 制作流程 Banner（新增，2026-03-25）

**内容**：展示 6 个工具的典型使用顺序（1→2→3→4→5→6），用步骤圆点 + 连接线串联。

**设计决策**：
- 解决"有了工具，不知道怎么用"的上手困惑
- 将 6 个独立工具串联为一条完整工作流，提升工具组合价值感
- 连接线使用品牌色渐变（`opacity: 0.2`），低调但有视觉引导

---

### 6. CTA Banner

**设计**：与工具详情页一致的品牌渐变背景，底部转化引导。

**文案**："一个人，也能做出专业级游戏资产"——直接回应独立游戏开发者的核心痛点（人手不足）。

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 点击分类 Tab | 过滤工具网格（客户端 JS 过滤） | SSG 场景下避免多页 |
| 悬浮工具卡片 | 卡片升起 + 阴影增强 + 鼠标指针变化 | 明确可点击性 |
| 点击工具卡 | 跳转到工具详情页 `/tools/[slug]` | 页面跳转 |
| 点击 Hero「立即免费试用」 | 进入工作台（根据访问来源自动选择合适工具） | 快速体验 |
| 滚动页面 | Sticky Nav 始终可见 | 导航可及性 |

## 可访问性说明

- 工具卡片使用 `<article>` 语义化标签。
- 过滤 Tab 使用 `role="tablist"` 和 `role="tab"`，支持键盘 Arrow 键切换。
- 搜索框有明确的 `aria-label="搜索工具"`（功能暂未实现，预留）。
- 卡片中的 Badge 使用 `aria-label` 描述含义（如 `aria-label="热门工具"`）。
