# 登录 / 注册页设计说明

**原型文件**：`design/pages/auth-design.html`  
**主题**：Dark Theme（深色主题）  
**渲染策略**：CSR（客户端渲染）  
**路由**：`/auth/login`、`/auth/register`（同一页面，Tab 切换）  
**最后更新**：2026-03-24  

---

## 设计目标

认证页是用户正式进入产品的入口，承担两个核心任务：

1. **降低注册/登录摩擦**：OAuth 一键授权优先，邮箱表单作为备选，整体流程不超过 3 步
2. **强化「进入产品」的仪式感**：用左侧视觉区展示产品的创作成果，让用户在注册前就感受到产品价值

目标用户：首次注册的新用户 + 回访老用户。新老用户走的路径不同（Register vs Login Tab），但视觉体验完全一致。

---

## 主题策略

认证页采用 **Dark Theme（深色主题）**，设计规范中明确规定：

> 认证页（登录/注册）采用深色主题，强化"进入产品"的仪式感。

理由：认证页是「公域营销页」和「产品内页」之间的过渡区，用深色主题让用户从首页浅色过渡到工作台深色，形成自然的心理暗示——「我即将进入创作空间」。

左侧略窄（5:6），让表单区域有足够空间，避免压迫感。

## 区块结构与设计决策

### 1. 左侧视觉面板（Visual Panel）

**内容**：品牌 Logo + 作品网格预览 + 底部 Slogan + 核心数据

**设计决策**：
- 背景使用与首页 Hero 相同的「多光晕」渐变（`radial-gradient`），而非插图，保持品牌语言一致性
- 叠加 40px 网格线纹理（`background-image` 细线），增加「技术感」背景细节，不喧宾夺主
- 作品网格使用 CSS 渐变色块模拟 AI 生成图（2列×3行，首张图跨行占更大视觉比重），配色对应不同工具类型（紫/蓝/绿/黄/红），上线后替换为真实图
- 每张作品卡片右下角带工具标签（半透明品牌色胶囊），让用户能「认出」工具
- 底部 Slogan 简洁直接，数据指标（50万+用户 / 2000万+作品 / 免费起步）在注册前再次建立信任

**与设计规范关联**：`--gradient-brand`、`--bg-surface`、`--border-subtle`、`--color-accent-*`

---

### 2. 右侧表单面板（Form Panel）

#### 2a. 返回首页链接

**设计决策**：
- 右上角固定位置，小字低调，给用户「反悔」的出口
- 带左箭头图标，符合「后退」的视觉认知

---

#### 2b. 标题区（Auth Header）

**内容**：动态标题 + 副标题（含切换链接）

**设计决策**：
- 标题根据当前模式动态切换：登录 → 「欢迎回来」，注册 → 「创建账号」
- 副标题附带切换链接（「还没有账号？免费注册」↔「已有账号？立即登录」），让用户在登录/注册之间自由跳转，不必通过 Tab

---

#### 2c. 模式切换 Tab

**内容**：登录 / 注册两档 Tab

**设计决策**：
- 使用「药丸形容器 + 内嵌高亮块」的 Tab 样式，而非下划线 Tab——在深色背景下，内嵌高亮块的视觉效果更清晰
- 背景色：`--bg-elevated`（比页面稍亮），高亮块：`--bg-overlay`（再亮一级），层次分明
- Tab 切换时，下方表单区域平滑切换（`display` 切换 + JS），OAuth 按钮保持不变（登录/注册都支持）

---

#### 2d. OAuth 按钮

**内容**：微信登录 + Google 登录，两个独立按钮

**设计决策**：
- 两个 OAuth 方式各占一行，而非并排——更大的点击区域，移动端更友好
- 微信按钮使用微信绿（`#07C160`）图标，Google 按钮使用 Google 官方四色图标，满足品牌合规要求
- 按钮样式：`--bg-elevated` 底色 + `--border-default` 边框，hover 时背景加深，视觉权重低于主 CTA（邮箱表单的提交按钮）
- **微信排在 Google 之前**：面向国内用户，微信是主流登录方式

---

#### 2e. 分割线

**设计决策**：
- 「或使用邮箱」分割线将 OAuth 和表单分区，视觉上明确两种登录方式的优先级（OAuth 优先）
- 分割线颜色极淡（`--border-subtle`），避免成为视觉干扰

---

#### 2f. 登录表单

**字段**：邮箱 + 密码（含显示/隐藏切换）+ 忘记密码链接

**设计决策**：
- 「忘记密码」放在密码字段标签旁，右对齐——用户寻找忘记密码的路径最短
- 密码可见性切换按钮内嵌在输入框右侧，减少布局跳动
- 输入框聚焦时：边框换为品牌色 + `box-shadow` 光晕，视觉上强化「当前输入焦点」

---

#### 2g. 注册表单

**字段**：邮箱 + 密码（含强度指示）+ 确认密码 + 服务条款勾选

**设计决策**：
- **密码强度指示器**：4格横条，随输入实时更新（弱/中/强），颜色分别用红/黄/绿——帮助用户主动设置强密码，减少后续账号安全问题
- 强度判断规则：长度≥8 / 含大写字母 / 含数字 / 含特殊字符，每满足一条+1格
- **服务条款勾选**：内联 Checkbox，勾选才能注册，满足法律合规要求
- Checkbox 样式自定义：选中后背景换为品牌渐变色 + 白色对勾，比浏览器默认样式更美观

---

#### 2h. 提交按钮

**设计决策**：
- 全宽品牌渐变按钮，是整个右侧面板视觉权重最高的元素，引导用户完成行动
- 登录按钮文案「登录」，注册按钮文案「免费注册」——「免费」二字再次降低注册心理门槛
- 提交后按钮进入 Loading 状态（旋转圆圈 + 文案变「登录中/注册中」），给用户明确的操作反馈
- 成功后弹出 Toast 提示，3 秒后自消失

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| Tab 切换（登录↔注册） | 表单区域切换，标题/副标题同步更新 | OAuth 按钮区域不随 Tab 变化 |
| 副标题链接点击 | 等同于点击对应 Tab | 双路径切换，符合用户直觉 |
| 密码显示/隐藏 | input type 切换 text/password，图标变化 | 每个密码输入框独立控制 |
| 密码实时强度检测 | 4格指示条颜色实时更新 | 仅注册表单有，登录不显示 |
| OAuth 按钮点击 | 跳转第三方授权页（原型中显示 Toast 模拟） | 实际跳转至微信/Google OAuth |
| 表单提交 | 按钮进入 Loading 状态 1.8s，再弹 Toast | 原型模拟，实际对接后端接口 |
| Toast 通知 | 顶部中间滑入，3秒后自动消失 | 成功/错误状态均使用此组件 |

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥769px | 左右分栏，左侧视觉区 + 右侧表单区 |
| ≤768px | 左侧视觉区完全隐藏，右侧表单全屏居中显示 |

移动端隐藏左侧视觉区，原因：手机屏幕空间有限，左侧纯视觉区会压缩表单区域；用户在手机上注册时更专注于完成表单，不需要视觉说服。

## 可访问性说明

- 所有 `<input>` 有对应 `<label>`（显式 `for/id` 关联）
- 密码可见性切换按钮有 `aria-label="显示/隐藏密码"`
- 自定义 Checkbox 保留原生 `<input type="checkbox">`，支持键盘空格键操作
- 提交按钮 Loading 状态有 `disabled` 防重复提交
- 色彩对比度：主文字 `#F0F0F8` on `#0D0D12` ≈ 18:1（远超 WCAG AA）
- 支持 Tab 键顺序导航（按顺序：Tab → OAuth → 表单字段 → 提交按钮）
