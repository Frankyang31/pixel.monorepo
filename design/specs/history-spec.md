# 我的作品集 设计说明

**原型文件**：`history-design.html`  
**主题**：Dark Theme  
**渲染策略**：CSR（需登录态，用户私有数据）  
**路由**：`/history`  
**最后更新**：2026-03-24

---

## 设计目标

我的作品集是用户管理和回顾自己 AI 创作记录的核心私人空间，目标是**帮助用户快速找到历史作品、管理作品状态（公开/私密）、复用历史参数继续创作**。核心路径：进入作品集 → 筛选/搜索 → 查看作品详情 → 复用参数回到工作台 OR 分享到广场。

---

## 主题策略

Dark Theme，与工作台、账户系列保持一致，属于"登录后产品内页"的视觉体系。深色调与 AI 生成图片的色彩表现相得益彰，避免浅色背景稀释图片视觉效果。

---

## 区块结构与设计决策

### 1. 顶部导航（已登录版）

**内容**：Logo + 产品内导航（工作台/广场/我的）+ 积分余额 Badge + 生成按钮 + 用户头像。

**设计决策**：
- 已登录状态导航与公域页不同，突出"回到工作台"的操作路径。
- 积分余额常驻展示，提醒用户当前可用资源。

**与设计规范关联**：`--accent-amber`、`--bg-elevated`、`--border-default`

---

### 2. 左侧侧边栏（账户导航）

**内容**：我的作品集（当前）/ 个人中心 / 积分明细 / 会员管理。

**设计决策**：
- 账户系列页面（作品集/个人中心/积分/会员）共用同一左侧导航，减少用户在账户区域的导航学习成本。
- 侧边栏宽度 200px，item 采用 icon + 文字，`active` 态使用品牌色浅背景高亮。
- 移动端侧边栏折叠为顶部 Tab 导航。

**与设计规范关联**：`--brand-start`、`--bg-elevated`、`--radius-md`

---

### 3. 页头工具栏

**内容**：页面标题 + 视图切换（网格/列表）+ 筛选器（按类型/日期/公开状态）+ 排序下拉 + 搜索框。

**设计决策**：
- 视图切换（网格/列表）满足不同用户偏好，网格适合浏览，列表适合管理。
- 筛选和排序靠右对齐，与内容区左对齐形成层次感。
- 搜索框允许按 Prompt 关键词搜索历史作品，对 Prompt 较多的重度用户极有价值。

**与设计规范关联**：`--bg-surface`、`--border-default`、`--radius-md`

---

### 4. 作品网格（主内容区）

**内容**：每个作品卡包含：缩略图、生成时间、工具类型标签、公开/私密状态、更多操作菜单（复用参数/分享/删除）。

**设计决策**：
- 每行 4~5 列（宽屏），比广场的 5 列略少，给操作元素留出更多空间。
- 公开/私密状态使用 Toggle 开关直接操作，无需跳转详情页，降低操作成本。
- 更多操作（⋯）菜单包含高频操作：复用参数（回到工作台）、下载、分享、删除。
- 批量选择模式：点击"选择"进入多选模式，支持批量删除/批量公开。

**与设计规范关联**：`--bg-surface`、`--shadow-md`、`--radius-xl`

---

### 5. 空状态

**内容**：无作品时展示引导插图 + 文案 + "立即创作"按钮。

**设计决策**：
- 空状态是新用户首次进入此页的必经场景，需要将空状态转化为引导创作的机会。
- 插图使用品牌渐变色调，与整体设计一致。

---

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 切换视图 | 网格/列表视图切换 | CSS 类切换 |
| 公开/私密 Toggle | 即时修改作品可见性 | API 调用，乐观更新 |
| 点击"复用参数" | 携带历史参数跳转工作台 | `?from=history&taskId=xxx` |
| 批量选择 | 进入多选模式，底部出现批量操作栏 | JS 状态管理 |
| 删除作品 | 显示确认对话框，确认后删除 | Modal 确认 |

---

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥1280px | 侧边栏 200px + 主区 4 列网格 |
| 1024-1279px | 侧边栏收窄 or 折叠，主区 3 列 |
| 768-1023px | 侧边栏折叠为顶部 Tab，主区 3 列 |
| <768px | 顶部 Tab，主区 2 列 |

---

## 可访问性说明

- 作品卡的公开/私密 Toggle 有 `aria-label="切换作品可见性"`。
- 批量选择模式中，Checkbox 组件使用标准 `<input type="checkbox">` 实现。
- 删除确认对话框使用 `role="dialog"` 和 `aria-modal="true"`，焦点锁定在对话框内。
