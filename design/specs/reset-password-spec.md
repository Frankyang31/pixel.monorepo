# 找回密码 设计说明

**原型文件**：`reset-password-design.html`  
**主题**：Dark Theme  
**渲染策略**：CSR（动态交互，无 SEO 需求）  
**路由**：`/reset-password`  
**最后更新**：2026-03-24

---

## 设计目标

找回密码流程的核心目标是**以最少的步骤帮助用户重新获得账号访问权**，同时保障账号安全。设计原则：简洁、清晰、减少挫败感（忘密码本身已让用户心情不好）。核心路径：输入邮箱 → 收到验证码 → 输入验证码 → 设置新密码 → 成功 → 跳转登录。

---

## 主题策略

Dark Theme，与登录/注册页保持一致，属于"认证流程"页面体系。同一视觉体系让用户在身份验证流程中感受一致的品牌体验。

---

## 区块结构与设计决策

### 1. 页面布局（左右分栏）

**内容**：左侧品牌视觉区（渐变背景 + Logo + 价值标语）+ 右侧步骤表单区。

**设计决策**：
- 复用登录/注册页的双栏布局，保持认证流程视觉一致性，减少用户认知切换成本。
- 左侧品牌区在找回密码这一负面场景下起到情绪稳定作用，传递"我们就在这里"的安全感。
- 移动端折叠为单列，仅展示表单区（左侧品牌区隐藏），聚焦核心任务。

**与设计规范关联**：`--bg-base`、`--bg-surface`、`--brand-start`、`--brand-end`

---

### 2. 步骤指示器

**内容**：3 步圆点指示器（步骤 1: 验证邮箱 → 步骤 2: 输入验证码 → 步骤 3: 设置新密码），含已完成 ✓ 样式和进行中状态。

**设计决策**：
- 步骤指示器让用户清楚知道"还差几步"，降低流程中途放弃的概率。
- 已完成步骤显示品牌色填充 + ✓ 图标，当前步骤为活跃高亮，未来步骤为灰色占位。
- 步骤间连接线随进度填充，视觉进度更直观。
- 3 步以内的流程无需更复杂的步骤条，圆点样式足够清晰。

**与设计规范关联**：`--brand-start`、`--text-tertiary`、`--border-default`

---

### 3. 步骤 1：输入邮箱

**内容**：邮箱输入框 + "发送验证码"按钮 + 返回登录链接。

**设计决策**：
- 仅有一个输入字段，认知负担最低。
- 发送后按钮变为"已发送（60s 后重试）"倒计时状态，防止刷单和用户多次点击。
- "返回登录"链接放在表单下方，让操作优先级清晰（主路径 > 回退）。

**与设计规范关联**：`--bg-elevated`、`--border-default`、`--radius-xl`

---

### 4. 步骤 2：输入验证码

**内容**：6 位验证码独立输入框（每格一个数字）+ 倒计时重发 + 切换到密码找回方式。

**设计决策**：
- 6 格独立输入框（OTP 风格）比单行输入框更直观，用户填写时有明确的"完成感"。
- 每个格子宽度较大，移动端友好，减少输入错误。
- 自动聚焦下一格：前一格输入完毕后，焦点自动移至下一格（JS 实现）。
- 重发倒计时 60 秒，结束后才能重发，防止滥用。

**与设计规范关联**：`--bg-elevated`、`--border-brand`（聚焦态）、`--radius-xl`

---

### 5. 步骤 3：设置新密码

**内容**：新密码输入框 + 确认密码输入框 + 密码强度指示器 + 提交按钮。

**设计决策**：
- 密码强度指示器实时反馈（弱/中/强），鼓励用户设置高强度密码。
- 确认密码框在提交时校验一致性，不一致时行内即时提示（不等待提交）。
- 密码可见切换图标（👁）默认隐藏，点击切换，减少输入错误。

**与设计规范关联**：`--accent-rose`（弱）、`--accent-amber`（中）、`--accent-teal`（强）

---

### 6. 成功状态

**内容**：大号成功图标 + 祝贺文案 + "前往登录"按钮。

**设计决策**：
- 成功态通过 JS class 切换展示（无页面跳转），用户视觉上有"完成"的成就感。
- 文案保持简洁轻松（"密码已重置，欢迎回来！"），消解找回密码过程的负面情绪。
- 3 秒后自动跳转登录页，也提供手动点击按钮（避免用户等待焦虑）。

**与设计规范关联**：`--accent-teal`、`--brand-start`

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 点击"发送验证码" | 按钮变为倒计时 60s，禁用再点击 | `setInterval` |
| 验证码格子输入 | 自动聚焦下一格 | `input` 事件监听 |
| 粘贴验证码 | 自动分拆填充 6 个格子 | `paste` 事件处理 |
| 密码输入实时 | 更新密码强度指示器 | JS 正则强度检测 |
| 提交新密码 | 校验通过后切换到成功页 | JS class 切换 |
| 成功页计时 | 3 秒后自动跳转 `/auth` | `setTimeout` |

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥1024px | 左右双栏（品牌区 + 表单区各 50%） |
| <1024px | 单列，隐藏左侧品牌区，表单居中 |
| <768px | 表单全宽，步骤指示器略缩小 |

---

## 可访问性说明

- 验证码 6 格使用 `<input type="tel" inputmode="numeric" maxlength="1">` 确保手机弹出数字键盘。
- 密码强度指示器有 `aria-live="polite"` 属性，实时播报强度变化。
- 每个步骤切换时焦点自动移至步骤标题或第一个输入框，避免焦点丢失。
- 错误提示使用 `role="alert"` 确保屏幕阅读器立即播报。
