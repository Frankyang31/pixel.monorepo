# 作品详情页 设计说明

**原型文件**：`gallery-detail-design.html`  
**主题**：Dark Theme  
**渲染策略**：SSR（需实时数据，同时有 SEO 价值）  
**路由**：`/gallery/[id]`  
**最后更新**：2026-03-24

---

## 设计目标

作品详情页承担三个核心职责：①让用户沉浸式欣赏单张作品；②透明展示生成参数，激励用户"复刻/改进"；③通过评论和相关推荐延长用户停留，深化社区互动。核心路径：点击广场卡片 → 查看大图 → 了解参数 → 点赞/评论 → 点击"用同款参数生成"回到工作台。

---

## 主题策略

Dark Theme，原因与广场一致：暗色背景让 AI 生成图片呈现最优视觉效果，颜色更鲜艳，对比度更高。详情页是用户最深度欣赏作品的场所，暗色营造沉浸式"画廊感"。

---

## 区块结构与设计决策

### 1. 顶部导航（公域版）

**内容**：Logo + 主导航 + 登录/注册按钮，保持简洁不干扰观图。

---

### 2. 面包屑导航

**内容**：首页 › 作品广场 › 作品标题，帮助用户清晰定位当前位置。

**设计决策**：
- 面包屑字号小、颜色浅（`--text-tertiary`），不抢夺主内容注意力，但提供清晰的回退路径。

**与设计规范关联**：`--text-tertiary`

---

### 3. 双栏布局（左图 + 右信息）

**内容**：左侧为图片查看器（占主要视觉空间），右侧为创作者信息、标题、操作按钮、生成参数。

**设计决策**：
- 黄金比例约 6:4（左:右），左侧图片为绝对主角，右侧信息支持决策。
- 左侧图片区域 `position: sticky`，随用户查看右侧长内容时图片保持在视口，避免失去参照感。
- 图片容器使用 1:1 正方形（与大多数 AI 生图默认比例一致），特殊比例（16:9 等）自适应。

**与设计规范关联**：`--bg-surface`、`--radius-2xl`

---

### 4. 缩略图切换栏（同一批次多图）

**内容**：主图下方的小缩略图横排，点击切换主图展示。

**设计决策**：
- AI 生成任务一次可生成 4 张变体，缩略图切换让用户一次性查看全部变体。
- 活跃缩略图使用品牌色边框（2px solid `--brand-start`）标识。
- 缩略图宽度固定 72px，超出时横向滚动。

**与设计规范关联**：`--brand-start`、`--radius-lg`

---

### 5. 创作者行

**内容**：头像 + 用户名 + 粉丝/作品数 + 关注按钮。

**设计决策**：
- 关注按钮使用描边样式（非填充），视觉重量低于下方的"下载"等主操作。
- 已关注状态变为"已关注"灰色，减少视觉干扰。

---

### 6. 标题、描述与标签

**内容**：作品标题（大号）+ 作者描述（1~2 句）+ 风格/内容标签（可点击过滤）。

**设计决策**：
- 标题使用 24px font-weight 800，仅次于导航区的视觉层级。
- 标签可点击，跳转到广场对应标签过滤结果，增加站内链接密度（SEO 价值）。

---

### 7. 数据统计行 + 操作按钮

**内容**：浏览量/点赞数/评论数/下载量，操作：点赞、分享、下载原图。

**设计决策**：
- 统计数字四维度并列显示，不做图表，简洁直观。
- "下载原图"按钮使用全宽渐变样式（最高视觉权重），因为这是对内容消费者最高价值的操作。
- 点赞按钮点击有即时动效（图标缩放），提供操作反馈。

**与设计规范关联**：`--brand-start`、`--brand-end`、`--accent-rose`

---

### 8. 复刻 CTA（"用同款参数生成"）

**内容**：全宽按钮，品牌色浅背景 + 品牌色文字，引导用户到工作台复用参数。

**设计决策**：
- 这是将"浏览用户"转化为"创作用户"的核心路径，布局位置紧邻生成参数卡上方，"看完参数就能马上用"的设计动线。
- 使用浅背景而非实心渐变，避免与"下载"按钮抢夺注意力（下载是消费者路径，复刻是创作者路径）。

**与设计规范关联**：`--border-brand`、`--brand-start`、`--bg-elevated`

---

### 9. 生成参数卡

**内容**：Prompt / 负向 Prompt / 模型 / 尺寸 / 步数 / CFG / 采样器 / Seed / 生成时间，一键复制 Prompt。

**设计决策**：
- 参数完整透明展示是平台与竞品差异化的核心点之一（参考 Civitai 做法）。
- "复制 Prompt"按钮在 Prompt 字段下方，位置明确，方便复制后直接去用。
- 参数卡使用表格形式（键值对），左列灰色 label，右列白色值，层次清晰。

**与设计规范关联**：`--bg-surface`、`--text-tertiary`、`--text-primary`

---

### 10. 评论区

**内容**：评论输入框（需登录）+ 评论列表（头像/名字/时间/内容/点赞/回复）。

**设计决策**：
- 评论区放在生成参数卡下方，先看作品，再了解参数，再参与讨论——符合用户自然行为顺序。
- 评论点赞/回复等操作需登录，未登录用户点击触发注册引导。

---

### 11. 相关作品推荐（页面底部）

**内容**：同一作者更多作品（4 格） + 相似风格推荐（4 格）。

**设计决策**：
- 两组推荐分开标题区分，避免用户混淆。
- 4 列网格与广场保持一致，降低学习成本。

---

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 点击缩略图 | 切换主图显示 | JS activeClass 切换 |
| 全屏按钮 | 主图全屏展示 | CSS/JS 全屏 API |
| 点击点赞 | 图标动效 + 计数 +1 | 需登录，乐观更新 |
| 点击复制 Prompt | 复制到剪贴板，按钮变为"✅ 已复制" 2 秒后还原 | Clipboard API |
| 发表评论 | 提交评论，列表顶部追加 | API 调用 |

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥1280px | 左右 6:4 双栏，左图 sticky |
| 1024-1279px | 双栏，左图 sticky |
| 768-1023px | 单列：图片 full-width → 信息区 → 参数 → 评论 |
| <768px | 单列，图片全宽，相关推荐 2 列 |

---

## 可访问性说明

- 主图有完整 `alt`（作品标题 + Prompt 前 80 字符）。
- 点赞按钮 `aria-pressed="true/false"` 反映状态。
- 评论区输入框有 `aria-label="发表评论"`。
- 缩略图切换有 `aria-label="查看第 N 张变体图"`。
