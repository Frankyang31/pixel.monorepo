# 个人中心 设计说明

**原型文件**：`account-design.html`  
**主题**：Dark Theme  
**渲染策略**：CSR（需登录态，用户私有数据）  
**路由**：`/account`  
**最后更新**：2026-03-24

---

## 设计目标

个人中心是用户管理账号信息、查看当前订阅状态和账户安全设置的统一入口，目标是**让用户快速完成信息修改、了解当前权益状态，并在合适位置引导升级会员**。核心路径：查看/编辑基本信息 → 确认会员权益 → 修改安全设置 → 管理通知偏好。

---

## 主题策略

Dark Theme，与账户系列其他页（作品集/积分/会员）保持一致，属于"登录后产品内页"体系。

---

## 区块结构与设计决策

### 1. 顶部导航 + 左侧侧边栏

**内容**：与账户系列共享导航和侧边栏结构（详见 history-spec.md）。

**与设计规范关联**：参见 history-spec.md

---

### 2. 个人资料卡（主区顶部）

**内容**：大号用户头像（可上传）、用户名、注册邮箱、注册时间、会员 Badge、"编辑资料"按钮。

**设计决策**：
- 头像区域使用渐变背景圆形容器，未上传时显示名字首字母占位（符合主流 SaaS 设计规范）。
- 会员 Badge 使用高亮样式（Pro / 免费）紧贴用户名展示，常驻提醒当前权益层级。
- "编辑资料"按钮触发 Modal 内联表单，无需跳转新页面，降低操作打断感。

**与设计规范关联**：`--bg-surface`、`--radius-2xl`、`--brand-start`、`--accent-amber`

---

### 3. 账号信息 Section

**内容**：邮箱（可修改）、绑定手机号（可修改/绑定）、已绑定的第三方账号（微信/Google）。

**设计决策**：
- 每一行信息右侧有"修改"按钮，点击后展开内联表单（不跳转新页面）。
- 第三方登录绑定状态使用绿色 "✓" 标识，未绑定显示"绑定"按钮（引导多端互通）。
- 安全敏感操作（修改邮箱/手机号）需要二次验证提示。

**与设计规范关联**：`--bg-elevated`、`--border-default`、`--radius-xl`

---

### 4. 安全设置 Section

**内容**：修改密码入口、登录设备管理（最近登录设备列表 + 一键退出其他设备）、二步验证（可选）。

**设计决策**：
- 密码修改不展示旧密码字段（减少泄露风险），而是通过邮件验证码方式重置。
- 设备管理列表显示设备类型图标、IP 地理位置、最后活跃时间，帮助用户发现异常登录。
- 二步验证 MVP 阶段可标注"即将上线"，占位留空间不影响整体布局。

**与设计规范关联**：`--accent-rose`、`--bg-surface`

---

### 5. 通知偏好 Section

**内容**：邮件通知 Toggle（产品动态/活动/任务完成提醒）、站内通知 Toggle。

**设计决策**：
- Toggle 开关即时生效（乐观更新），无需"保存"按钮，降低操作摩擦。
- 通知类型分组清晰，用户可精细控制，减少因"垃圾邮件"导致的取消订阅。

**与设计规范关联**：`--brand-start`、`--border-default`

---

### 6. 危险区域（账号注销）

**内容**：注销账号入口（高亮警告样式），点击弹出二次确认对话框。

**设计决策**：
- 注销入口在页面最底部，使用红色高亮区块（`--accent-rose`）和警告文案，避免误操作。
- 二次确认对话框要求用户手动输入"注销"二字确认，双重防护。
- 注销流程说明数据处理方式（90 天内删除），符合《个人信息保护法》要求。

**与设计规范关联**：`--accent-rose`、`--bg-surface`

## 交互细节

| 交互 | 行为 | 说明 |
|------|------|------|
| 点击头像 | 弹出上传头像对话框（裁剪） | Modal |
| 点击"修改"（邮箱/手机） | 展开内联表单 + 发送验证码 | CSS 折叠动画 |
| Toggle 通知设置 | 即时生效，乐观更新 | API 调用 |
| 点击"退出其他设备" | 确认后踢出其他 session | API 调用 |
| 点击"注销账号" | 弹出确认对话框 | Modal + 文字确认 |

## 响应式策略

| 断点 | 布局变化 |
|------|----------|
| ≥1280px | 侧边栏 + 主区单列（最大宽度 760px 居中） |
| 1024-1279px | 侧边栏折叠，主区单列 |
| 768-1023px | 顶部 Tab 导航，主区全宽 |
| <768px | 顶部 Tab，全宽单列 |

---

## 可访问性说明

- 表单字段均有关联 `<label>`，确保屏幕阅读器可正确读取。
- Toggle 开关使用 `role="switch"` 和 `aria-checked` 状态。
- 危险区域注销按钮有 `aria-describedby` 关联警告文案。
- 确认对话框使用焦点陷阱（`focus-trap`），键盘用户可正常操作。
