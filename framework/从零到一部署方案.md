## 一、基础设施选型与账号准备

### 1.1 需要注册/申请的账号（部分有审核周期，最早开始）

| 服务 | 用途 | 审核周期 | 优先级 |
|------|------|---------|--------|
| **微信开放平台** | 微信 OAuth 登录（国内用户） | 3-7 个工作日 | 🔴 最高，第一天就申请 |
| **微信商户平台** | 微信支付收款 | 1-5 个工作日 | 🔴 最高，第一天就申请 |
| **Stripe** | 海外支付收款 | 实名认证 1-3 天 | 🔴 最高 |
| **Google Cloud Console** | Google OAuth 登录（海外用户）| 即时，但需配置 | 🟡 较高 |
| **Cloudflare** | DNS + CDN + R2 对象存储 | 即时 | 🟡 较高 |
| **GitHub** | 代码托管 + CI/CD | 即时 | 🟢 普通 |
| **Sentry** | 错误监控 | 即时 | 🟢 普通 |
| **阿里云/腾讯云短信** | 手机号验证码 | 企业实名 1-3 天 | 🟡 较高 |

> ⚠️ **关键提醒**：微信支付和微信 OAuth 的资质审核经常在功能开发完后才开始申请，结果卡住上线。这两项务必在第一天并行推进，不依赖开发进度。

### 1.2 服务器选型

**推荐方案：Hetzner Cloud（德国/芬兰/美国节点）**

| 配置 | 型号 | 月费 | 适用场景 |
|------|------|------|---------|
| MVP 起步 | CX32（4核8G，160GB SSD）| ~€15.9/月（约 ¥125） | DAU < 500 |
| MVP 稳定期 | CX42（8核16G，240GB SSD）| ~€32.9/月（约 ¥255） | DAU 500-2000 |

**备选方案**（如果需要国内低延迟）：
- 腾讯云轻量应用服务器 4核8G，约 ¥200/月
- 阿里云轻量服务器 4核8G，约 ¥200/月

> 面向海外用户优先选 Hetzner（价格极佳）；国内用户为主选阿里云/腾讯云香港节点。

## 二、服务器配置

### 2.1 购买服务器后的初始化

```bash
# 以 Ubuntu 22.04 LTS 为例

# 1. 更新系统包
apt update && apt upgrade -y

# 2. 创建部署用户（不用 root 直接操作）
adduser deploy
usermod -aG sudo deploy

# 3. 配置 SSH 密钥登录（在本地执行）
ssh-copy-id -i ~/.ssh/id_rsa.pub deploy@<服务器IP>

# 4. 禁止 SSH 密码登录，只允许密钥
sudo sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo sed -i 's/PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo systemctl restart sshd

# 5. 配置防火墙（UFW）
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw deny 5432/tcp   # 不对外暴露 PostgreSQL
sudo ufw deny 6379/tcp   # 不对外暴露 Redis
sudo ufw enable

# 6. 设置时区
sudo timedatectl set-timezone Asia/Shanghai
```

### 2.2 安装 Docker 和 Docker Compose

```bash
# 安装 Docker（官方一键脚本）
curl -fsSL https://get.docker.com | sudo sh

# 把 deploy 用户加入 docker 组
sudo usermod -aG docker deploy

# 验证
docker --version       # Docker version 25.x.x
docker compose version # Docker Compose version v2.x.x

# 重新登录以使 docker 组生效
exit && ssh deploy@<服务器IP>
```

### 2.3 目录结构规划

```bash
mkdir -p /home/deploy/app          # 项目根目录
mkdir -p /home/deploy/data/postgres # PostgreSQL 数据持久化
mkdir -p /home/deploy/data/redis    # Redis 数据持久化
mkdir -p /home/deploy/logs          # 应用日志
mkdir -p /home/deploy/backups       # 数据库备份
mkdir -p /home/deploy/certs         # SSL 证书（Certbot 会自动管理）
```

## 三、域名与 DNS

### 3.1 域名购买建议

- **海外用户为主**：Cloudflare Registrar（成本价注册，无溢价）、Namecheap
- **国内用户为主**：阿里云万网、腾讯云（需 ICP 备案）
- **推荐格式**：`yourbrand.com`（主站）、`api.yourbrand.com`（API）、`assets.yourbrand.com`（CDN 资产）

> ⚠️ **ICP 备案**：国内服务器必须备案，域名才能解析到国内 IP。备案流程约 15-20 个工作日，优先级等同微信支付审核。

### 3.2 将域名接入 Cloudflare（强烈推荐）

1. 在 Cloudflare 注册账号，添加域名
2. 在域名注册商处，将 Nameserver 改为 Cloudflare 提供的两个 NS 地址
3. 等待 NS 生效（通常 5 分钟-1 小时）

Cloudflare 提供免费的 DNS 管理、DDoS 防护、CDN 加速，是性价比最高的选择。

### 3.3 DNS 记录配置

在 Cloudflare DNS 面板中添加：

| 类型 | 名称 | 内容 | 代理状态 |
|------|------|------|---------|
| A | `@`（主域名）| 服务器 IP | 已代理（橙色云朵）|
| A | `www` | 服务器 IP | 已代理 |
| A | `api` | 服务器 IP | 已代理 |
| CNAME | `assets` | `你的R2自定义域名.r2.dev` | 已代理 |

## 四、对象存储（Cloudflare R2）

### 4.1 创建 R2 存储桶

1. 登录 Cloudflare 控制台 → R2 对象存储 → 创建存储桶
2. 存储桶名称建议：`aigen-assets-prod`（生产）、`aigen-assets-dev`（开发）
3. 位置：选择离目标用户最近的区域（亚太选 `APAC`）

### 4.2 配置公开访问（CDN 域名绑定）

```
R2 存储桶设置 → 自定义域名 → 添加 assets.yourbrand.com
```

系统会自动在 Cloudflare DNS 中创建 CNAME 记录，并启用 CDN 加速。

### 4.3 创建 API 令牌

1. Cloudflare → 我的个人资料 → API 令牌 → 创建令牌
2. 选择「编辑 Cloudflare Workers」模板，或自定义以下权限：
   - 账户 → R2 存储 → 编辑
3. 记录以下三个值（只显示一次，立即保存到密码管理器）：
   ```
   CLOUDFLARE_ACCOUNT_ID=xxxx
   R2_ACCESS_KEY_ID=xxxx
   R2_SECRET_ACCESS_KEY=xxxx
   ```

### 4.4 存储桶文件夹结构规划

```
aigen-assets-prod/
├── images/
│   ├── {user_id}/
│   │   └── {task_id}.webp      # 用户生成的图片
├── videos/
│   └── {user_id}/
│       └── {task_id}.mp4
├── avatars/
│   └── {user_id}/
│       └── avatar.webp          # 用户头像
└── system/
    └── examples/                # 官方示例素材
```

### 4.5 CORS 策略配置

在 R2 存储桶 → 设置 → CORS 策略 中添加：

```json
[
  {
    "AllowedOrigins": [
      "https://yourbrand.com",
      "https://www.yourbrand.com",
      "http://localhost:3000"
    ],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedHeaders": ["Content-Type", "Authorization"],
    "MaxAgeSeconds": 3600
  }
]
```

## 五、CDN 配置

R2 绑定自定义域名后，Cloudflare CDN 会自动生效。以下是关键的缓存规则配置：

### 5.1 缓存规则（Page Rules）

在 Cloudflare → 规则 → 页面规则中添加：

**规则 1：AI 生成物长期缓存（不可变资源）**
```
URL 匹配：assets.yourbrand.com/images/*
设置：缓存级别 = 缓存所有内容，边缘缓存 TTL = 1 个月
```

**规则 2：视频资源缓存**
```
URL 匹配：assets.yourbrand.com/videos/*
设置：缓存级别 = 缓存所有内容，边缘缓存 TTL = 1 周
```

### 5.2 生成物的 HTTP 缓存头设置

在后端上传到 R2 时，设置正确的缓存头：

```python
# backend/app/core/storage/r2_client.py
import boto3
from botocore.config import Config

class R2Client:
    def __init__(self):
        self.client = boto3.client(
            "s3",
            endpoint_url=f"https://{CLOUDFLARE_ACCOUNT_ID}.r2.cloudflarestorage.com",
            aws_access_key_id=R2_ACCESS_KEY_ID,
            aws_secret_access_key=R2_SECRET_ACCESS_KEY,
            config=Config(signature_version="s3v4"),
        )
    
    async def upload_image(
        self, 
        file_path: str, 
        object_key: str,
        immutable: bool = True
    ) -> str:
        """
        上传图片到 R2
        - 生成物是不可变资源，设置长期缓存
        - 通过 URL 版本化（task_id）来区分不同版本
        """
        cache_control = (
            "public, max-age=31536000, immutable"  # 1 年，不可变
            if immutable
            else "public, max-age=86400"            # 普通资源 1 天
        )
        
        with open(file_path, "rb") as f:
            self.client.put_object(
                Bucket=R2_BUCKET_NAME,
                Key=object_key,
                Body=f,
                ContentType="image/webp",
                CacheControl=cache_control,
            )
        
        return f"https://assets.yourbrand.com/{object_key}"
```

## 六、数据库（PostgreSQL）

### 6.1 Docker Compose 中的 PostgreSQL 配置

```yaml
# docker-compose.yml（PostgreSQL 部分）
services:
  postgres:
    image: pgvector/pgvector:pg16   # 带 pgvector 扩展的官方镜像
    container_name: aigen_postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}    # 从 .env 注入，不写死
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_INITDB_ARGS: "--encoding=UTF8 --locale=C"
    volumes:
      - /home/deploy/data/postgres:/var/lib/postgresql/data
      - ./backend/db/init.sql:/docker-entrypoint-initdb.d/init.sql  # 初始化脚本
    ports:
      - "127.0.0.1:5432:5432"   # 只绑定本地回环，不对外暴露
    shm_size: 256m               # 提升 PostgreSQL 共享内存，避免大查询 OOM
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      interval: 10s
      timeout: 5s
      retries: 5
```

### 6.2 数据库初始化脚本

```sql
-- backend/db/init.sql

-- 启用 pgvector 扩展（RAG 场景备用）
CREATE EXTENSION IF NOT EXISTS vector;
-- 启用 UUID 生成
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 用户表
CREATE TABLE IF NOT EXISTS users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    email VARCHAR(255) UNIQUE,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255),
    nickname VARCHAR(100),
    avatar_url TEXT,
    oauth_provider VARCHAR(20),       -- 'wechat' | 'google' | null
    oauth_provider_id VARCHAR(255),
    membership_level SMALLINT NOT NULL DEFAULT 0,  -- 0=free, 1=basic, 2=pro, 3=enterprise
    membership_expires_at TIMESTAMPTZ,
    is_active BOOLEAN NOT NULL DEFAULT true,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 积分账户表
CREATE TABLE IF NOT EXISTS points_accounts (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    balance DECIMAL(18, 2) NOT NULL DEFAULT 0 CHECK (balance >= 0),
    frozen_balance DECIMAL(18, 2) NOT NULL DEFAULT 0 CHECK (frozen_balance >= 0),
    version INTEGER NOT NULL DEFAULT 0,          -- 乐观锁版本号
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 积分流水表
CREATE TABLE IF NOT EXISTS points_transactions (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES users(id),
    transaction_id VARCHAR(200) NOT NULL,       -- 幂等键（业务唯一标识）
    type VARCHAR(20) NOT NULL,                  -- EARN/SPEND/FREEZE/UNFREEZE/REFUND
    amount DECIMAL(18, 2) NOT NULL,
    balance_before DECIMAL(18, 2) NOT NULL,
    balance_after DECIMAL(18, 2) NOT NULL,
    source VARCHAR(50) NOT NULL,                -- MEMBERSHIP/SIGN/INVITE/ACTIVITY/API_CALL
    reference_id VARCHAR(100),                  -- 关联的任务ID/订单ID
    reference_type VARCHAR(50),                 -- 关联类型
    remark TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_transaction_id UNIQUE (transaction_id)  -- 数据库层幂等保证
);

-- AI 生成任务表
CREATE TABLE IF NOT EXISTS tasks (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES users(id),
    tool_slug VARCHAR(50) NOT NULL,             -- 工具标识（character-art/environment-art 等）
    model VARCHAR(100) NOT NULL,                -- 使用的 AI 模型
    prompt TEXT NOT NULL,
    negative_prompt TEXT,
    params JSONB,                               -- 额外参数（宽高、步数等）
    status VARCHAR(20) NOT NULL DEFAULT 'pending',  -- pending/queued/processing/completed/failed/cancelled
    celery_task_id VARCHAR(255),               -- Celery 任务 ID
    result_url TEXT,                           -- 生成结果 URL（R2 地址）
    error_message TEXT,
    frozen_points DECIMAL(18, 2),              -- 预扣积分数
    actual_points DECIMAL(18, 2),              -- 实际消耗积分数
    is_public BOOLEAN NOT NULL DEFAULT false,
    processing_started_at TIMESTAMPTZ,
    completed_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 会员订单表
CREATE TABLE IF NOT EXISTS membership_orders (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES users(id),
    plan_id VARCHAR(50) NOT NULL,              -- 套餐标识
    amount_cny DECIMAL(10, 2),                -- 人民币金额
    amount_usd DECIMAL(10, 2),                -- 美元金额
    currency VARCHAR(10) NOT NULL,             -- CNY/USD
    payment_provider VARCHAR(20) NOT NULL,     -- wechat/alipay/stripe
    payment_order_id VARCHAR(255),            -- 支付平台订单号
    status VARCHAR(20) NOT NULL DEFAULT 'pending',  -- pending/paid/refunded/failed
    points_granted DECIMAL(18, 2),            -- 到账积分数
    paid_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 常用索引
CREATE INDEX idx_tasks_user_status ON tasks(user_id, status);
CREATE INDEX idx_tasks_created ON tasks(created_at DESC);
CREATE INDEX idx_points_tx_user ON points_transactions(user_id, created_at DESC);
CREATE INDEX idx_users_email ON users(email) WHERE email IS NOT NULL;
CREATE INDEX idx_users_phone ON users(phone) WHERE phone IS NOT NULL;
CREATE INDEX idx_users_oauth ON users(oauth_provider, oauth_provider_id) WHERE oauth_provider IS NOT NULL;

-- updated_at 自动更新触发器函数
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_users_updated_at BEFORE UPDATE ON users
    FOR EACH ROW EXECUTE FUNCTION update_updated_at();

CREATE TRIGGER trg_points_accounts_updated_at BEFORE UPDATE ON points_accounts
    FOR EACH ROW EXECUTE FUNCTION update_updated_at();

CREATE TRIGGER trg_tasks_updated_at BEFORE UPDATE ON tasks
    FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

## 七、缓存与队列（Redis）

### 7.1 Docker Compose 中的 Redis 配置

```yaml
redis:
  image: redis:7-alpine
  container_name: aigen_redis
  restart: unless-stopped
  command: >
    redis-server
    --requirepass ${REDIS_PASSWORD}
    --maxmemory 1gb
    --maxmemory-policy allkeys-lru
    --save 900 1          # RDB：900秒内有1次写则持久化
    --save 300 10         # 300秒内有10次写
    --save 60 10000       # 60秒内有10000次写
    --appendonly yes      # AOF 持久化开启
    --appendfsync everysec
  volumes:
    - /home/deploy/data/redis:/data
  ports:
    - "127.0.0.1:6379:6379"   # 只绑定本地，不对外暴露
  healthcheck:
    test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
    interval: 10s
    timeout: 5s
    retries: 5
```

### 7.2 Redis Key 命名规范

```
# 任务状态
task:status:{task_id}                  → "pending/processing/completed/failed"
task:progress:{task_id}                → 进度百分比 (0-100)

# SSE 进度推送（任务执行时 Worker 写入，前端读取）
task:events:{task_id}                  → List，存储进度事件 JSON

# 积分余额（增长阶段 Redis 预扣方案启用）
points:balance:{user_id}               → 余额（Decimal 字符串）
points:frozen:{user_id}                → 冻结余额

# 熔断器状态
circuit:{model_id}:failures            → 失败次数（60s 窗口）
circuit:{model_id}:state               → "closed/open/half-open"

# 限流
ratelimit:{endpoint}:{user_id}         → 请求计数（1分钟窗口）

# 分布式锁
lock:points:deduct:{user_id}           → 1（存在即锁定）

# 缓存
cache:exact:{prompt_hash}              → 生成物 URL（24h TTL）

# 签到
sign:date:{user_id}                    → YYYYMMDD（今日是否已签到）
sign:ip:{ip_address}                   → Set，该 IP 下的用户 ID 集合
```

## 八、后端部署（FastAPI + Celery）

### 8.1 Dockerfile（后端）

```dockerfile
# backend/Dockerfile
FROM python:3.12-slim

# 设置工作目录
WORKDIR /app

# 安装系统依赖（仅构建所需）
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq-dev \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# 先复制依赖文件，利用 Docker 层缓存
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制源码
COPY . .

# 非 root 用户运行
RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /app
USER appuser

# 暴露端口
EXPOSE 8000

# 启动命令（由 docker-compose 的 command 覆盖以区分 API 和 Worker）
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

```nginx
        add_header Cache-Control "public, max-age=31536000, immutable";
    }
    
    # 页面请求（SSR）

        add_header X-Accel-Buffering "no";
```

## 十二、CI/CD 流水线

### 12.1 GitHub Actions 工作流

```yaml
# .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches:
      - main    # 推送 main 分支时触发自动部署

jobs:
  deploy:
    name: Deploy
    runs-on: ubuntu-latest
    
    steps:
      # 1. 拉取代码
      - name: Checkout
        uses: actions/checkout@v4
      
      # 2. 配置 SSH
      - name: Setup SSH
        uses: webfactory/ssh-agent@v0.9.0
        with:
          ssh-private-key: ${{ secrets.DEPLOY_SSH_KEY }}
      
      # 3. 添加服务器到 known_hosts（首次需要手动确认）
      - name: Add server to known_hosts
        run: |
          ssh-keyscan -H ${{ secrets.DEPLOY_HOST }} >> ~/.ssh/known_hosts
      
      # 4. 远程部署脚本
      - name: Deploy
        env:
          DEPLOY_HOST: ${{ secrets.DEPLOY_HOST }}
          DEPLOY_USER: ${{ secrets.DEPLOY_USER }}
        run: |
          ssh $DEPLOY_USER@$DEPLOY_HOST << 'EOF'
            set -e
            
            # 进入项目目录
            cd /home/deploy/app
            
            # 拉取最新代码
            git pull origin main
            
            # 重新构建并启动（--build 强制重新构建镜像）
            docker compose pull
            docker compose build --no-cache
            
            # 滚动更新：先启动新容器，再停止旧容器
            docker compose up -d --remove-orphans
            
            # 等待健康检查通过
            sleep 10
            docker compose ps
            
            # 运行数据库迁移（如果有）
            docker compose exec -T backend alembic upgrade head
            
            # 清理旧镜像
            docker image prune -f
            
            echo "✅ 部署完成：$(date)"
          EOF
      
      # 5. 部署失败时通知（可选，接入企业微信/钉钉 webhook）
      - name: Notify on failure
        if: failure()
        run: |
          curl -X POST "${{ secrets.NOTIFY_WEBHOOK }}" \
            -H "Content-Type: application/json" \
            -d '{"text": "⚠️ 生产环境部署失败，请立即检查！"}'
```

```python
# backend/app/api/v1/health.py
from fastapi import APIRouter
from sqlalchemy import text
from app.db.database import get_db
from app.core.redis_client import redis_client

router = APIRouter()

@router.get("/health")
async def health_check():
    """基础健康检查（Nginx 和 Docker 用）"""
    return {"status": "ok"}

@router.get("/health/deep")
async def deep_health_check():
    """深度健康检查（检查各依赖连通性）"""
    checks = {}
    
    # 检查数据库
    try:
        async with get_db() as db:
            await db.execute(text("SELECT 1"))
        checks["database"] = "ok"
    except Exception as e:
        checks["database"] = f"error: {str(e)}"
    
    # 检查 Redis
    try:
        await redis_client.ping()
        checks["redis"] = "ok"
    except Exception as e:
        checks["redis"] = f"error: {str(e)}"
    
    # 检查 Celery Workers（查看活跃 worker 数）
    try:
        from app.core.celery_app import celery_app
        inspect = celery_app.control.inspect(timeout=2)
        active = inspect.active()
        checks["celery"] = f"ok, {len(active or {})} worker(s)"
    except Exception as e:
        checks["celery"] = f"error: {str(e)}"
    
    all_ok = all(v == "ok" or v.startswith("ok,") for v in checks.values())
    return {
        "status": "ok" if all_ok else "degraded",
        "checks": checks
    }
```

```python
def configure_logging():
    structlog.configure(
        processors=[
            structlog.contextvars.merge_contextvars,
            structlog.stdlib.add_log_level,
            structlog.stdlib.add_logger_name,
            structlog.processors.TimeStamper(fmt="iso"),
            structlog.processors.StackInfoRenderer(),
            structlog.processors.format_exc_info,
            structlog.processors.JSONRenderer(),     # 输出 JSON 格式，方便日志收集
        ],
        wrapper_class=structlog.stdlib.BoundLogger,
        context_class=dict,
        logger_factory=structlog.stdlib.LoggerFactory(),
    )

# 在请求中间件中自动注入 request_id
@app.middleware("http")
async def request_id_middleware(request: Request, call_next):
    request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))
    structlog.contextvars.bind_contextvars(request_id=request_id)
    response = await call_next(request)
    response.headers["X-Request-ID"] = request_id
    return response
```

## 十四、备份策略

### 14.1 PostgreSQL 每日自动备份

```bash
# /home/deploy/scripts/backup_postgres.sh
#!/bin/bash

set -e

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/home/deploy/backups"
BACKUP_FILE="${BACKUP_DIR}/postgres_${DATE}.sql.gz"
CONTAINER="aigen_postgres"

# 执行备份（压缩）
docker exec "${CONTAINER}" pg_dump \
    -U "${POSTGRES_USER}" \
    "${POSTGRES_DB}" \
    | gzip > "${BACKUP_FILE}"

echo "✅ 备份完成：${BACKUP_FILE}（$(du -sh ${BACKUP_FILE} | cut -f1)）"

# 上传到 R2（需要 rclone 配置 R2 remote）
# rclone copy "${BACKUP_FILE}" r2:aigen-backups/postgres/

# 保留最近 14 天，删除旧备份
find "${BACKUP_DIR}" -name "postgres_*.sql.gz" -mtime +14 -delete

echo "🧹 旧备份清理完成"
```

```bash
# 设置可执行权限
chmod +x /home/deploy/scripts/backup_postgres.sh

# 注册 crontab（每天凌晨 3 点执行）

# 添加一行：
0 3 * * * /home/deploy/scripts/backup_postgres.sh >> /home/deploy/logs/backup.log 2>&1
```

## 十五、安全基线

### 15.1 服务器层面

```bash
# 安装 fail2ban（防暴力破解 SSH）
sudo apt install -y fail2ban

# 配置 fail2ban
sudo cat > /etc/fail2ban/jail.local << 'EOF'
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 5
bantime = 3600
findtime = 600
EOF

sudo systemctl enable fail2ban && sudo systemctl start fail2ban
```

### 15.2 应用层面

**环境变量安全**：
- `.env` 必须加入 `.gitignore`
- 生产服务器上，`.env` 文件权限设为 `600`（只有 deploy 用户可读）
- 定期轮换所有 API Key（建议每 90 天）

**敏感接口保护**：
```python
# FastAPI 中的内部接口需要 API Key 认证
from fastapi.security import APIKeyHeader

api_key_header = APIKeyHeader(name="X-Internal-API-Key")

async def verify_internal_key(api_key: str = Security(api_key_header)):
    if api_key != settings.INTERNAL_API_KEY:
        raise HTTPException(status_code=403, detail="Forbidden")
```

**Nginx 隐藏版本信息**：
```nginx
# /etc/nginx/nginx.conf
server_tokens off;
```

### 15.3 数据库层面

```bash
# PostgreSQL 和 Redis 均只绑定 127.0.0.1，禁止从外网直接连接
```
