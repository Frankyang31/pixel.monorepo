## 二、分层职责边界

系统采用严格的四层架构，**禁止跨层调用**：

```
HTTP 请求
    ↓
┌─────────────────────────────┐
│  API 路由层（api/v1/）       │  ← 只做：路由分发、Pydantic 校验、依赖注入、响应格式化
│  - 无业务逻辑                │
│  - 不直接操作数据库          │
└───────────────┬─────────────┘
                ↓ 调用 Service
┌─────────────────────────────┐
│  业务服务层（services/）     │  ← 只做：业务规则、事务编排、调用 Core/DB
│  - 无 HTTP 概念              │
│  - 跨模块协调（如扣费+入队）  │
└───────────────┬─────────────┘
                ↓ 调用 Core、Model
┌─────────────────────────────┐
│  核心基础设施层（core/）     │  ← 只做：技术能力封装（Redis、R2、模型调用、JWT）
│  - 无业务语义                │
│  - 可被多个 Service 复用     │
└───────────────┬─────────────┘
                ↓
┌─────────────────────────────┐
│  数据存储层（models/ + db/）  │  ← SQLAlchemy ORM + PostgreSQL
│  - 只暴露数据读写            │
└─────────────────────────────┘
```

### 典型调用链示例：提交生成任务

```
POST /api/v1/generation/jobs
  └── api/v1/generation.py::create_job()
        ├── 验证请求（Pydantic）
        ├── 注入 current_user（JWT Depends）
        └── 调用 generation_service.submit_job()
              ├── points_service.freeze()         # 预扣积分
              ├── db.execute(INSERT INTO tasks)   # 写任务记录
              ├── celery_app.send_task()          # 入队
              └── return JobResponse(jobId, ...)
```

## 三、完整数据库 Schema

> 在现有 `从零到一部署方案.md` 中 `init.sql` 基础上补充完整字段和约束。

```sql
-- ================================================================
-- 扩展
-- ================================================================
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
CREATE EXTENSION IF NOT EXISTS vector;  -- pgvector，RAG 场景备用

-- ================================================================
-- 用户表
-- ================================================================
CREATE TABLE users (
    id                   UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    -- 登录凭证（三选一或多选，至少绑定一个）
    email                VARCHAR(255) UNIQUE,
    phone                VARCHAR(20) UNIQUE,
    password_hash        VARCHAR(255),               -- bcrypt，为 NULL 表示纯 OAuth 注册
    -- OAuth
    oauth_wechat_openid  VARCHAR(128) UNIQUE,
    oauth_wechat_unionid VARCHAR(128),
    oauth_google_sub     VARCHAR(128) UNIQUE,
    -- 基本信息
    nickname             VARCHAR(100) NOT NULL DEFAULT '用户',
    avatar_url           TEXT,
    locale               VARCHAR(10) NOT NULL DEFAULT 'zh-CN',  -- zh-CN | en
    -- 会员状态（冗余字段，快速读取权益，通过 membership_orders 事件同步）
    membership_level     SMALLINT NOT NULL DEFAULT 0,          -- 0=free 1=basic 2=pro 3=enterprise
    membership_expires_at TIMESTAMPTZ,
    -- 安全
    is_active            BOOLEAN NOT NULL DEFAULT true,
    is_email_verified    BOOLEAN NOT NULL DEFAULT false,
    last_login_at        TIMESTAMPTZ,
    last_login_ip        INET,
    -- 时间戳
    created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at           TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_email          ON users(email) WHERE email IS NOT NULL;
CREATE INDEX idx_users_phone          ON users(phone) WHERE phone IS NOT NULL;
CREATE INDEX idx_users_wechat_openid  ON users(oauth_wechat_openid) WHERE oauth_wechat_openid IS NOT NULL;
CREATE INDEX idx_users_google_sub     ON users(oauth_google_sub) WHERE oauth_google_sub IS NOT NULL;

-- ================================================================
-- 积分账户表（每用户一行，是资金快照）
-- ================================================================
CREATE TABLE points_accounts (
    id             UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id        UUID NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    balance        DECIMAL(18, 2) NOT NULL DEFAULT 0 CHECK (balance >= 0),
    frozen_balance DECIMAL(18, 2) NOT NULL DEFAULT 0 CHECK (frozen_balance >= 0),
    total_earned   DECIMAL(18, 2) NOT NULL DEFAULT 0,   -- 历史总获取（对账用）
    total_spent    DECIMAL(18, 2) NOT NULL DEFAULT 0,   -- 历史总消耗
    version        INTEGER NOT NULL DEFAULT 0,           -- 乐观锁版本号
    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- ================================================================
-- 积分流水表（每次变动一行，审计核心）
-- ================================================================
CREATE TABLE points_transactions (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id         UUID NOT NULL REFERENCES users(id),
    -- 幂等保障
    transaction_id  VARCHAR(200) NOT NULL,   -- 业务唯一键，如 DEDUCT:job_abc123
    -- 变动信息
    type            VARCHAR(20) NOT NULL,    -- EARN | SPEND | FREEZE | UNFREEZE | REFUND
    amount          DECIMAL(18, 2) NOT NULL, -- 正数（方向由 type 决定）
    balance_before  DECIMAL(18, 2) NOT NULL,
    balance_after   DECIMAL(18, 2) NOT NULL,
    -- 来源
    source          VARCHAR(50) NOT NULL,    -- SIGNUP_BONUS | SIGN_IN | MEMBERSHIP | API_CALL | ACTIVITY | REFUND
    reference_id    VARCHAR(200),            -- 关联业务 ID（任务 ID、订单 ID 等）
    reference_type  VARCHAR(50),             -- job | order | activity
    remark          TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT uq_points_transaction_id UNIQUE (transaction_id)
);

CREATE INDEX idx_points_tx_user_time ON points_transactions(user_id, created_at DESC);
CREATE INDEX idx_points_tx_ref       ON points_transactions(reference_id) WHERE reference_id IS NOT NULL;

-- ================================================================
-- AI 生成任务表（job，对应前端的"任务"概念）
-- ================================================================
CREATE TABLE generation_jobs (
    id               UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id          UUID NOT NULL REFERENCES users(id),
    -- 工具与参数
    tool_slug        VARCHAR(50) NOT NULL,    -- character-art | environment-art | sprite-sheet | ...
    model_provider   VARCHAR(50) NOT NULL,    -- openai | tongyi | stability
    model_name       VARCHAR(100) NOT NULL,   -- dall-e-3 | wanx-v1 | stable-diffusion-xl
    prompt           TEXT NOT NULL,
    negative_prompt  TEXT,
    aspect_ratio     VARCHAR(10) NOT NULL DEFAULT '2:3',  -- 1:1 | 2:3 | 3:4 | 9:16 | 16:9 | 21:9
    style_preset     VARCHAR(50),             -- 奇幻 RPG | 二次元 | 像素游戏 | 写实风格 | 赛博朋克
    reference_image_url TEXT,                 -- img2img / inpainting 参考图（R2 URL）
    extra_params     JSONB DEFAULT '{}',      -- 其他扩展参数（steps、cfg_scale 等）
    -- 状态
    status           VARCHAR(20) NOT NULL DEFAULT 'submitted',
                     -- submitted → queued → processing → done | failed | cancelled
    celery_task_id   VARCHAR(255),            -- Celery task ID（入队后写入）
    progress         SMALLINT NOT NULL DEFAULT 0,  -- 0–100
    error_message    TEXT,
    -- 积分
    credits_frozen   DECIMAL(18, 2),          -- 预扣积分数
    credits_charged  DECIMAL(18, 2),          -- 实际消耗积分数（完成后写入）
    credits_refunded DECIMAL(18, 2) DEFAULT 0,
    -- 时间
    queued_at        TIMESTAMPTZ,
    processing_started_at TIMESTAMPTZ,
    completed_at     TIMESTAMPTZ,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_jobs_user_status   ON generation_jobs(user_id, status);
CREATE INDEX idx_jobs_user_created  ON generation_jobs(user_id, created_at DESC);
CREATE INDEX idx_jobs_celery_id     ON generation_jobs(celery_task_id) WHERE celery_task_id IS NOT NULL;
CREATE INDEX idx_jobs_status_queued ON generation_jobs(created_at) WHERE status = 'queued';

-- ================================================================
-- 资产表（asset，任务完成后生成，与 job 1:1 或 1:N，
--         sprite-sheet 一次任务生成多张图，可 1:N）
-- ================================================================
CREATE TABLE assets (
    id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    job_id       UUID NOT NULL REFERENCES generation_jobs(id) ON DELETE CASCADE,
    user_id      UUID NOT NULL REFERENCES users(id),
    -- 文件信息
    storage_key  VARCHAR(500) NOT NULL,    -- R2 对象 key（如 images/{user_id}/{job_id}.webp）
    thumb_key    VARCHAR(500),             -- 缩略图 key
    file_format  VARCHAR(20) NOT NULL DEFAULT 'webp',  -- webp | png | mp4
    width        INTEGER,
    height       INTEGER,
    file_size    INTEGER,                  -- 字节
    -- 展示属性
    is_public    BOOLEAN NOT NULL DEFAULT false,
    is_starred   BOOLEAN NOT NULL DEFAULT false,
    likes_count  INTEGER NOT NULL DEFAULT 0,
    -- 时间
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_assets_user         ON assets(user_id, created_at DESC);
CREATE INDEX idx_assets_user_starred ON assets(user_id) WHERE is_starred = true;
CREATE INDEX idx_assets_public       ON assets(created_at DESC) WHERE is_public = true;
CREATE INDEX idx_assets_job          ON assets(job_id);

-- ================================================================
-- 资产点赞表（防重复点赞）
-- ================================================================
CREATE TABLE asset_likes (
    asset_id   UUID NOT NULL REFERENCES assets(id) ON DELETE CASCADE,
    user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (asset_id, user_id)
);

-- ================================================================
-- 会员订单表
-- ================================================================
CREATE TABLE membership_orders (
    id                UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id           UUID NOT NULL REFERENCES users(id),
    -- 套餐
    plan_id           VARCHAR(50) NOT NULL,         -- basic_monthly | pro_monthly | pro_annual | ...
    plan_name         VARCHAR(100) NOT NULL,
    credits_granted   DECIMAL(18, 2) NOT NULL,       -- 本次订单赠送积分数
    membership_level  SMALLINT NOT NULL,
    membership_days   INTEGER NOT NULL,              -- 授权天数
    -- 金额
    currency          VARCHAR(10) NOT NULL,          -- CNY | USD
    amount            DECIMAL(10, 2) NOT NULL,
    -- 支付
    payment_provider  VARCHAR(20) NOT NULL,          -- wechat | alipay | stripe
    payment_order_id  VARCHAR(255),                  -- 支付平台订单号（第三方返回）
    payment_intent_id VARCHAR(255),                  -- Stripe Payment Intent ID
    -- 状态
    status            VARCHAR(20) NOT NULL DEFAULT 'pending',
                      -- pending | paid | refunded | expired | cancelled
    paid_at           TIMESTAMPTZ,
    -- 时间
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_orders_user        ON membership_orders(user_id, created_at DESC);
CREATE INDEX idx_orders_payment_id  ON membership_orders(payment_order_id) WHERE payment_order_id IS NOT NULL;
CREATE UNIQUE INDEX idx_orders_stripe_intent ON membership_orders(payment_intent_id) WHERE payment_intent_id IS NOT NULL;

-- ================================================================
-- 工具配置表（MVP 阶段可以写死代码，但建议数据库化方便运营更新）
-- ================================================================
CREATE TABLE tools (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    slug            VARCHAR(50) NOT NULL UNIQUE,   -- character-art | environment-art | ...
    name_zh         VARCHAR(100) NOT NULL,
    name_en         VARCHAR(100) NOT NULL,
    description_zh  TEXT,
    description_en  TEXT,
    icon_url        TEXT,
    -- 积分定价
    credits_min     SMALLINT NOT NULL,             -- 最少消耗
    credits_max     SMALLINT NOT NULL,             -- 最多消耗
    -- 模型配置（JSON，灵活配置不同会员等级对应的模型）
    model_config    JSONB NOT NULL DEFAULT '{}',
    -- 支持的参数
    supported_aspect_ratios  TEXT[] DEFAULT ARRAY['1:1','2:3','3:4','9:16'],
    supported_style_presets  TEXT[] DEFAULT ARRAY[]::TEXT[],
    supports_reference_image BOOLEAN NOT NULL DEFAULT false,
    supports_negative_prompt BOOLEAN NOT NULL DEFAULT true,
    -- 状态
    is_active       BOOLEAN NOT NULL DEFAULT true,
    sort_order      SMALLINT NOT NULL DEFAULT 0,   -- 展示排序
    -- SEO
    seo_title_zh    VARCHAR(200),
    seo_title_en    VARCHAR(200),
    seo_desc_zh     TEXT,
    seo_desc_en     TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- ================================================================
-- Refresh Token 表（支持多设备登录 + 主动吊销）
-- ================================================================
CREATE TABLE refresh_tokens (
    id          UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash  VARCHAR(255) NOT NULL UNIQUE,    -- SHA-256 哈希存储，不存明文
    device_info TEXT,                            -- UA / 设备标识
    ip_address  INET,
    is_revoked  BOOLEAN NOT NULL DEFAULT false,
    expires_at  TIMESTAMPTZ NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_refresh_tokens_user    ON refresh_tokens(user_id);
CREATE INDEX idx_refresh_tokens_active  ON refresh_tokens(user_id) WHERE is_revoked = false;

-- ================================================================
-- 通用 updated_at 自动更新触发器
-- ================================================================
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = NOW(); RETURN NEW; END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_users_updated_at             BEFORE UPDATE ON users             FOR EACH ROW EXECUTE FUNCTION set_updated_at();
CREATE TRIGGER trg_points_accounts_updated_at   BEFORE UPDATE ON points_accounts   FOR EACH ROW EXECUTE FUNCTION set_updated_at();
CREATE TRIGGER trg_generation_jobs_updated_at   BEFORE UPDATE ON generation_jobs   FOR EACH ROW EXECUTE FUNCTION set_updated_at();
CREATE TRIGGER trg_assets_updated_at            BEFORE UPDATE ON assets            FOR EACH ROW EXECUTE FUNCTION set_updated_at();
CREATE TRIGGER trg_membership_orders_updated_at BEFORE UPDATE ON membership_orders FOR EACH ROW EXECUTE FUNCTION set_updated_at();
CREATE TRIGGER trg_tools_updated_at             BEFORE UPDATE ON tools             FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

## 四、Redis Key 规范

所有 Key 遵循 `domain:sub_domain:identifier` 三段命名规范，禁止使用无前缀的裸 Key。

```
# ── 任务进度（Worker 写，API 读）──────────────────────
```

### 6.2 积分扣减（事务边界）

```python
# services/points_service.py

async def freeze_for_job(
    db: AsyncSession,
    redis: Redis,
    user_id: str,
    job_id: str,
    amount: Decimal
) -> None:
    """
    预扣积分 - 整个流程在单一数据库事务内完成
    Redis 分布式锁在事务外层获取，事务提交后释放
    """
    lock_key = f"lock:points:{user_id}"
    transaction_id = f"FREEZE:{job_id}"

    # 1. 获取分布式锁（最外层，序列化同一用户的并发操作）
    async with redis.lock(lock_key, timeout=30):
        async with db.begin():  # 开启数据库事务
            # 2. 幂等检查
            existing = await db.scalar(
                select(PointsTransaction).where(
                    PointsTransaction.transaction_id == transaction_id
                )
            )
            if existing:
                return  # 已处理，幂等返回

            # 3. 查询账户（加行锁，防并发）
            account = await db.scalar(
                select(PointsAccount)
                .where(PointsAccount.user_id == user_id)
                .with_for_update()
            )
            if not account or account.balance < amount:
                raise InsufficientCreditsError(
                    f"余额不足，当前 {account.balance}，需要 {amount}"
                )

            # 4. 更新余额（乐观锁版本号）
            updated = await db.execute(
                update(PointsAccount)
                .where(
                    PointsAccount.user_id == user_id,
                    PointsAccount.version == account.version  # 乐观锁
                )
                .values(
                    balance=account.balance - amount,
                    frozen_balance=account.frozen_balance + amount,
                    version=account.version + 1
                )
            )
            if updated.rowcount == 0:
                raise ConcurrentModificationError("并发冲突，请重试")

            # 5. 写流水（与余额更新在同一事务）
            db.add(PointsTransaction(
                user_id=user_id,
                transaction_id=transaction_id,
                type="FREEZE",
                amount=amount,
                balance_before=account.balance,
                balance_after=account.balance - amount,
                source="API_CALL",
                reference_id=job_id,
                reference_type="job"
            ))
            # 事务提交 ↑（db.begin() 的 __aexit__ 触发 COMMIT）
```

```python
# api/v1/webhooks.py

@router.post("/webhooks/wechat-pay")
async def wechat_pay_callback(
    request: Request,
    db: AsyncSession = Depends(get_db),
    redis: Redis = Depends(get_redis)
):
    """
    回调幂等保障：
    1. 验证微信签名（防伪造）
    2. 以 payment_order_id 为幂等键，防止重复处理
    3. 所有状态变更在同一事务内完成
    4. 向微信回复 200 OK（否则微信会重试）
    """
    body = await request.body()
    
    # 1. 验签（HMAC-SHA256）
    if not verify_wechat_signature(request.headers, body, settings.WECHAT_PAY_API_V3_KEY):
        raise HTTPException(400, "签名验证失败")

    payload = parse_wechat_notify(body, settings.WECHAT_PAY_API_V3_KEY)
    payment_order_id = payload["out_trade_no"]       # 我方订单号
    wechat_transaction_id = payload["transaction_id"]  # 微信流水号

    # 2. 幂等锁（防微信重试导致重复发放积分）
    idempotency_key = f"pay:done:{payment_order_id}"
    if not await redis.set(idempotency_key, "1", nx=True, ex=3600):
        return {"code": "SUCCESS"}  # 已处理，直接 ACK

    # 3. 事务内处理：更新订单 + 发放积分 + 更新会员
    await payment_service.confirm_payment(
        db=db,
        payment_order_id=payment_order_id,
        wechat_transaction_id=wechat_transaction_id,
    )

    return {"code": "SUCCESS"}
```

### 7.1 Token 体系

采用 **Access Token（短效）+ Refresh Token（长效）** 双 Token 机制：

| Token 类型 | 有效期 | 存储 | 用途 |
|-----------|--------|------|------|
| Access Token | 2 小时 | 内存（前端 JS 变量）| 请求鉴权 |
| Refresh Token | 30 天 | HttpOnly Cookie | 无感刷新 Access Token |

```python
# core/auth/jwt.py

ACCESS_TOKEN_EXPIRE = timedelta(hours=2)
REFRESH_TOKEN_EXPIRE = timedelta(days=30)

def create_access_token(user_id: str, membership_level: int) -> str:
    return jwt.encode({
        "sub": user_id,
        "ml": membership_level,     # membership_level，避免每次查库
        "type": "access",
        "exp": datetime.utcnow() + ACCESS_TOKEN_EXPIRE,
        "iat": datetime.utcnow(),
    }, settings.JWT_SECRET_KEY, algorithm="HS256")

def create_refresh_token(user_id: str) -> tuple[str, str]:
    """返回 (token_plain, token_hash)，数据库存 hash，返回给客户端 plain"""
    token = secrets.token_urlsafe(48)
    token_hash = hashlib.sha256(token.encode()).hexdigest()
    return token, token_hash
```

```python
    credentials: HTTPAuthorizationCredentials = Security(HTTPBearer()),
    db: AsyncSession = Depends(get_db)
```

### 8.2 熔断器实现

```python
# core/model_gateway/circuit_breaker.py

class CircuitBreaker:
    """
    三态熔断器：closed（正常）→ open（熔断）→ half_open（探测）→ closed
    状态存储在 Redis，多 Worker 共享
    """
    FAILURE_THRESHOLD = 5       # 60s 内失败 5 次则熔断
    FAILURE_WINDOW    = 60      # 失败计数窗口（秒）
    OPEN_DURATION     = 300     # 熔断持续时间（秒）
    HALF_OPEN_PROBE   = 1       # 半开状态允许通过的请求数

    def __init__(self, provider: str, redis: Redis):
        self.provider = provider
        self.redis = redis
        self._failure_key   = f"circuit:{provider}:failures"
        self._state_key     = f"circuit:{provider}:state"
        self._halfopen_key  = f"circuit:{provider}:halfopen_count"

    async def get_state(self) -> str:
        state = await self.redis.get(self._state_key)
        return (state or b"closed").decode()

    async def call(self, func, *args, **kwargs):
        state = await self.get_state()

        if state == "open":
            raise CircuitOpenError(f"模型 {self.provider} 熔断中，请稍后重试")

        if state == "half_open":
            probe_count = await self.redis.incr(self._halfopen_key)
            if probe_count > self.HALF_OPEN_PROBE:
                raise CircuitOpenError(f"模型 {self.provider} 探测中，请稍后重试")

        try:
            result = await func(*args, **kwargs)
            # 成功：重置失败计数，关闭熔断
            await self.redis.delete(self._failure_key, self._state_key, self._halfopen_key)
            return result
        except (RetryableError, asyncio.TimeoutError) as e:
            failures = await self.redis.incr(self._failure_key)
            await self.redis.expire(self._failure_key, self.FAILURE_WINDOW)

            if failures >= self.FAILURE_THRESHOLD:
                await self.redis.set(self._state_key, "open", ex=self.OPEN_DURATION)
                logger.warning("circuit_breaker_opened", provider=self.provider, failures=failures)

            raise
```

### 核心图像任务实现

```python
# core/task_queue/tasks/image_tasks.py

@celery_app.task(
    name="tasks.image.generate",
    bind=True,
    max_retries=5,
    autoretry_for=(RetryableModelError,),
    retry_backoff=True,          # 指数退避：1s, 2s, 4s, 8s, 16s
    retry_backoff_max=120,       # 最大等待 2 分钟
    retry_jitter=True,           # 随机抖动，防雷群效应
)
def generate_image(self, job_id: str):
    """
    图像生成 Worker 任务
    执行顺序：取任务 → 更新状态 → 调用模型 → 压缩上传 → 写 asset → 确认积分
    """
    with get_sync_db() as db:
        job = db.get(GenerationJob, job_id)
        if not job or job.status == "cancelled":
            return  # 任务已取消，直接退出

        try:
            # 1. 更新状态：processing
            job.status = "processing"
            job.processing_started_at = datetime.utcnow()
            db.commit()

            # 2. 写进度到 Redis（前端轮询读取）
            redis.set(f"job:progress:{job_id}", 10, ex=3600)
            redis.set(f"job:status:{job_id}", "processing", ex=3600)

            # 3. 获取模型适配器，调用 AI 生成
            adapter = get_adapter(job.model_provider, job.model_name)
            result_url = adapter.generate_sync(GenerateRequest(
                prompt=job.prompt,
                negative_prompt=job.negative_prompt,
                width=parse_width(job.aspect_ratio),
                height=parse_height(job.aspect_ratio),
                style_preset=job.style_preset,
                reference_url=job.reference_image_url,
            ))

            redis.set(f"job:progress:{job_id}", 70, ex=3600)

            # 4. 下载 → 压缩 → 上传 R2
            local_path = download_to_temp(result_url)
            webp_path, thumb_path = compress_to_webp(local_path)
            storage_key = f"images/{job.user_id}/{job_id}.webp"
            thumb_key   = f"thumbs/{job.user_id}/{job_id}.webp"
            r2.upload(webp_path, storage_key)
            r2.upload(thumb_path, thumb_key)

            redis.set(f"job:progress:{job_id}", 90, ex=3600)

            # 5. 写 assets 表
            asset = Asset(
                job_id=job_id,
                user_id=job.user_id,
                storage_key=storage_key,
                thumb_key=thumb_key,
                width=get_width(webp_path),
                height=get_height(webp_path),
                file_size=os.path.getsize(webp_path),
            )
            db.add(asset)

            # 6. 确认积分扣费（冻结 → 实际扣除）
            actual_cost = get_actual_cost(job.tool_slug, job.model_name)
            points_service.confirm_freeze(
                db=db,
                user_id=job.user_id,
                job_id=job_id,
                actual_amount=actual_cost
            )

            # 7. 更新任务完成
            job.status = "done"
            job.credits_charged = actual_cost
            job.completed_at = datetime.utcnow()
            db.commit()

            # 8. 清除进度缓存
            redis.delete(f"job:progress:{job_id}", f"job:status:{job_id}")

        except NonRetryableError as e:
            # 不可重试错误（如 prompt 违规）：直接标记失败，退还积分
            _handle_job_failure(db, job, str(e), refund=True)
            raise  # 不重试

        except Exception as e:
            # 可重试错误或 max_retries 耗尽后的最终失败
            if self.request.retries >= self.max_retries:
                _handle_job_failure(db, job, str(e), refund=True)
            raise  # 触发 Celery 重试机制
```

### 10.3 文件上传约束

上传参考图（用于 img2img / inpainting）的约束：

```python
UPLOAD_CONFIG = {
    "max_size_bytes": 10 * 1024 * 1024,    # 10 MB
    "allowed_types": ["image/jpeg", "image/png", "image/webp"],
    "max_width": 4096,
    "max_height": 4096,
}

@router.post("/assets/upload-reference")
async def upload_reference_image(
    file: UploadFile,
    current_user: User = Depends(get_current_user),
):
    # 校验文件类型（读 magic bytes，不信任 Content-Type header）
    header = await file.read(16)
    await file.seek(0)
    mime = magic.from_buffer(header, mime=True)
    if mime not in UPLOAD_CONFIG["allowed_types"]:
        raise HTTPException(400, "不支持的文件类型")
    
    # 校验文件大小
    contents = await file.read()
    if len(contents) > UPLOAD_CONFIG["max_size_bytes"]:
        raise HTTPException(400, "文件大小超过 10MB 限制")
    
    # 上传到 R2 的临时目录（24h 后自动清理）
    key = f"temp/{current_user.id}/{uuid4()}.{mime.split('/')[1]}"
    r2.upload_bytes(contents, key, content_type=mime)
    
    return {"url": f"{settings.R2_PUBLIC_DOMAIN}/{key}"}
```

## 十一、错误处理规范

### 11.1 异常层级

```python
# 自定义异常层级（app/utils/exceptions.py）

class AppError(Exception):
    """所有业务异常的基类"""
    http_status: int = 500
    error_code: str = "INTERNAL_ERROR"
    
    def __init__(self, message: str = ""):
        self.message = message

class AuthError(AppError):
    http_status = 401
    error_code = "UNAUTHORIZED"

class PermissionError(AppError):
    http_status = 403
    error_code = "FORBIDDEN"

class NotFoundError(AppError):
    http_status = 404
    error_code = "NOT_FOUND"

class InsufficientCreditsError(AppError):
    http_status = 422
    error_code = "INSUFFICIENT_CREDITS"

class RateLimitError(AppError):
    http_status = 429
    error_code = "RATE_LIMITED"

class RetryableModelError(AppError):
    """AI 模型可重试错误（Celery 会自动重试）"""
    pass

class NonRetryableModelError(AppError):
    """AI 模型不可重试错误（直接标记失败）"""
    pass
```

### 11.2 全局异常处理器

```python
# app/main.py

@app.exception_handler(AppError)
async def app_error_handler(request: Request, exc: AppError):
    logger.warning("app_error", code=exc.error_code, message=exc.message)
    return JSONResponse(
        status_code=exc.http_status,
        content={"code": exc.error_code, "message": exc.message, "detail": None}
    )

@app.exception_handler(RequestValidationError)
async def validation_error_handler(request: Request, exc: RequestValidationError):
    return JSONResponse(
        status_code=400,
        content={
            "code": "VALIDATION_ERROR",
            "message": "请求参数校验失败",
            "detail": exc.errors()
        }
    )

@app.exception_handler(Exception)
async def unhandled_exception_handler(request: Request, exc: Exception):
    logger.error("unhandled_exception", exc_info=exc)
    # Sentry 自动捕获
    return JSONResponse(
        status_code=500,
        content={"code": "INTERNAL_ERROR", "message": "服务器内部错误", "detail": None}
    )
```

## 十二、安全规范

### 12.1 密码与 Token 安全

| 对象 | 存储方式 | 说明 |
|------|---------|------|
| 用户密码 | bcrypt（rounds=12）| 禁止 MD5/SHA1 |
| Refresh Token | SHA-256 哈希后存 DB | 数据库泄露不暴露原始 token |
| API Key（内部服务）| SHA-256 哈希后存 DB | 同上 |
| 微信支付证书私钥 | 文件系统（600权限）+ 环境变量路径 | 不存 DB，不 Git 提交 |
| 第三方 API Key | `.env`（600权限）| 不 Git 提交 |

### 12.2 SQL 注入防御

全面使用 SQLAlchemy ORM 或参数化查询，**禁止字符串拼接 SQL**：

```python
# ❌ 错误示例（SQL 注入风险）
query = f"SELECT * FROM users WHERE email = '{email}'"

# ✅ 正确示例（参数化）
result = await db.execute(
    select(User).where(User.email == email)
)
```

### 12.3 Prompt 注入防御

用户 Prompt 不直接拼接进系统提示词，通过模板隔离：

```python
SYSTEM_PROMPT_TEMPLATE = """
你是一个游戏资产生成助手，请根据用户描述生成对应的图像提示词。
规则：只能生成与游戏资产相关的内容，拒绝生成任何违规内容。
"""

def build_model_prompt(tool_slug: str, user_prompt: str) -> str:
    """用户输入只填入 USER 消息，不污染 SYSTEM 提示词"""
    return {
        "system": SYSTEM_PROMPT_TEMPLATE,
        "user": user_prompt[:2000]  # 截断防超长
    }
```
