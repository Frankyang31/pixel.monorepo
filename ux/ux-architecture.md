# UX 架构建议文档

> 整理于 2026-03-25，为 AI 内容生成平台提供完整的技术 UX 基础框架。

---

## 一、布局架构

### 1.1 栅格系统

桌面端采用 12 列栅格，最大宽度 1280px，列间距 24px，左右边距 40px。平板端采用 8 列栅格，列间距 20px，左右边距 24px。移动端采用 4 列栅格，列间距 16px，左右边距 16px。

### 1.2 工作台布局

工作台采用经典的双栏结构：左侧为参数配置面板（固定宽度 320-360px），右侧为结果展示区（flex-1 自适应）。左侧面板使用 `position: sticky` 实现固定不随内容滚动，右侧结果区独立滚动支持无限加载历史内容。移动端折叠为上下布局，参数区默认收起。

```
┌────────────────────────────────────────────────────┐
│  Nav（64px 固定顶部）                               │
├──────────────┬─────────────────────────────────────┤
│              │                                     │
│  左侧面板     │  右侧结果区                          │
│  (320-360px) │  (flex-1)                           │
│              │                                     │
│  · 模式切换  │  · 生成中：任务占位卡片               │
│  · Prompt 区 │  · 完成：图片/视频网格展示            │
│  · 参数配置  │  · 支持多任务并行展示                 │
│  · 生成按钮  │                                     │
│              │                                     │
└──────────────┴─────────────────────────────────────┘
```

### 1.3 容器系统

移动端采用全宽设计，16px 内边距。平板端最大宽度 768px 居中。桌面端最大宽度 1024px 居中。大屏场景最大宽度 1280px 居中。

## 二、组件层级

组件按优先级分为四个层级：布局组件负责容器、网格、区块的基础结构；内容组件承载卡片、文章、媒体等内容展示；交互组件处理按钮、表单、导航等用户操作；工具组件提供间距、字体、颜色等辅助功能。

---

## 三、主题切换系统

### 3.1 双主题策略

公域营销页（首页、工具详情页、博客、定价页）使用浅色主题建立信任感，便于 SEO 内容阅读。产品内页（工作台、作品广场、个人中心）使用深色主题降低视觉疲劳，突出 AI 生成内容。认证页（登录/注册）采用深色主题强化进入产品的仪式感。

### 3.2 浅色主题色彩变量

背景层次方面，底层背景使用极浅暖灰 #F5F4F8，卡片面板使用纯白 #FFFFFF，悬浮元素同样使用纯白配合阴影区分，输入框和选中区域使用 #F0EFF4。边框分为四个透明度级别，分别用于分隔线、卡片、聚焦和品牌选中状态。文字颜色区分为主标题 #18181F、正文描述 #5A5A72、次要信息 #9090A8、禁用状态 #C0C0D0。功能色分别为成功 #0F9480、警告 #D97706、错误 #DC2626、信息 #2563EB。

### 3.3 深色主题色彩变量

背景层次采用近黑带微紫的 #0D0D12 作为底层，卡片使用 #13131A，悬浮元素使用 #1C1C27，输入框和选中区域使用 #252535。文字颜色分别对应主标题 #F0F0F8、次要信息 #A0A0B8、占位符 #6060785、禁用 #404058。功能色调整为成功 #2DD4BF、警告 #FBBF24、错误 #F87171、信息 #60A5FA。

### 3.4 主题切换组件

主题切换使用三个按钮的组合（浅色/深色/系统），通过 data-theme 属性控制 html 元素的样式。JavaScript 实现需要检测系统偏好、管理 localStorage 存储、实时应用主题切换并更新按钮选中状态。

## 四、响应式策略

采用移动优先的策略，基础设计从 320px 以上开始。768px 以上进入平板端增强布局。1024px 以上提供完整功能和内容。1280px 以上进行大屏优化。

### 断点设置

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| sm | < 640px | 单列布局，导航折叠 |
| md | 640-1024px | 双列布局，保留侧边 |
| lg | 1024-1280px | 完整布局 |
| xl | > 1280px | 居中约束，最大 1280px |

## 五、可访问性基础

### 5.1 键盘导航

所有交互元素需要支持 Tab 键聚焦，使用 :focus-visible 状态显示品牌色轮廓。确保合理的标签顺序，通过 tabindex 属性管理焦点顺序。模态框弹出时需要锁定焦点在模态框内，关闭时恢复焦点位置。

### 5.2 屏幕阅读器支持

使用语义化 HTML 元素（header、nav、main、section、article）。为非语义元素添加 ARIA 角色属性。为图标按钮提供 aria-label 描述。确保表单标签与输入框正确关联。

### 5.3 色彩对比度

文字与背景的对比度需要达到 WCAG 2.1 AA 标准的 4.5:1。大号文字（18px+ 或 14px+ 粗体）的对比度需要达到 3:1。功能色（成功、警告、错误）需要额外验证对比度是否符合要求。

### 5.4 动效偏好

检测 prefers-reduced-motion媒体查询，关闭所有非必要动画。功能性动效（加载状态、展开收起）保持启用，仅移除装饰性动效。过渡时长遵循设计规范：即时 0ms、快速 100ms、常规 200ms、慢速 300ms。

## 六、交互模式

### 6.1 导航交互

页面内锚点使用平滑滚动，导航激活状态通过 URL 变化自动更新。移动端采用汉堡菜单展开导航。

### 6.2 表单交互

输入框获得焦点时显示品牌色边框，验证错误时显示错误色并显示错误提示信息。提交按钮在表单验证通过前禁用。

### 6.3 按钮状态

所有按钮需要实现 hover、active、focus-visible 三种状态。hover 时轻微变色或上浮，active 时按下效果，focus-visible 时显示品牌色轮廓。

### 6.4 卡片交互

悬停时卡片上浮并出现品牌色光晕边框，增强点击反馈。卡片内的链接和按钮独立处理焦点状态。

## 七、文件结构建议

### CSS 文件组织

```
css/
├── design-system.css    # 变量和令牌（包含主题系统）
├── layout.css          # 网格和容器系统
├── components.css      # 可复用组件样式
├── utilities.css       # 辅助类和工具
└── main.css            # 项目特定覆盖
```

### JavaScript 文件组织

```
js/
├── theme-manager.js     # 主题切换功能
├── components.js        # 交互组件逻辑
└── main.js             # 项目特定 JavaScript
```

## 八、实施优先级

### 第一阶段：基础设置

首先实施设计系统变量，建立 CSS 自定义属性体系。然后创建响应式容器和网格系统，确保基础布局在不同设备上正常工作。

### 第二阶段：组件构建

构建可复用组件模板，包括按钮、表单、卡片、模态框等基础组件。实现主题切换功能，确保浅色/深色主题正确应用。

### 第三阶段：内容集成

添加具有适当层级的实际内容（H1 > H2 > H3 结构）。集成交互逻辑和状态管理。

### 第四阶段：交互润色

实施悬停状态、动画效果、过渡效果。优化可访问性支持，添加 ARIA 属性和键盘导航。

## 九、关键设计决策

### 工作台模式标签

工作台顶部的模式切换从「文生图/图生视频/风格迁移」调整为六个游戏资产生成工具：角色立绘、场景原画、Sprite 批量、动画预览、材质纹理、细节精修。每个工具增加对应的图标和颜色标识，便于用户识别。

### Prompt 占位符

Prompt 输入框的占位符更新为游戏角色和场景描述示例，符合游戏资产生成的产品定位。参数区域增加风格预设选项，包括奇幻 RPG、二次元、像素等游戏风格。

### 右侧结果区

右侧结果区的标题从通用的「生成结果」改为「我的角色库」「我的场景库」等与当前工具对应的名称，强化工具的垂类定位。

## 十、验证清单

在实施过程中需要验证以下要点：主题切换在所有页面正常工作且无闪烁。所有交互元素可通过键盘完整操作。响应式布局在 320px 至 1920px 范围内表现正常。色彩对比度符合 WCAG AA 标准。动效在检测到 reduced-motion 时正确关闭。
